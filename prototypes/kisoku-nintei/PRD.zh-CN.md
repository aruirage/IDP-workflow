# NeosAI 日本保险 IDP 工作流配置 — 产品需求文档（PRD）

## 第 6 章｜功能详细说明

### 6.01 业务场景设置



#### 6.01.1 概述

四步向导：Step1 案件集约规则，Step2 工作流画布，Step3 通知设置，Step4 输出字段与发布。新建、编辑已发布、复制场景共用流程。Step2 测试只是辅助检查，不再作为进入 Step3 和 Step4 的强制门禁。

#### 6.01.2 功能点

1. 场景列表：新建、复制、重命名、删除、关键词检索；复制后进入 Step1 继续编辑。
2. Step1：配置名称、说明、账票集合（1～20）、主账票与主字段、关联账票字段关系、匹配策略；支持关联检查与 AI 自动关联。
3. Step2：进入工作流画布；画布读取 Step1 账票与关系，限制节点可选范围。工具栏提供工作流测试；测试只用于辅助发现结构、连线和可达性问题，不再强制阻断进入 Step3。
4. Step3：按工作流节点配置内置通知。配置对象、主题、内容和可插入变量；页面不展示触发条件，系统按节点专属 `Status / Result` 变量自动判断。
5. Step4：从 OCR 抽出账票类型中勾选要导出的字段（允许 0）。Step2 未测通也可进入，但公开前须满足可发布条件。点保存固化勾选，不重跑画布测试。
6. 发布：场景须为可发布；有 OCR 候选时还须已保存输出勾选；无候选时可直接公开。新案件只读取已发布版本。

| 显示状态 | 实现取值        | 触发条件                                                                                      | 按钮影响                                                                       |

Step3：配置节点内置通知，不设启用/停用开关；配置存在即启用，删除配置即停用。点击「保存」或「次へ」都会保存当前通知配置；其中「次へ」在保存后直接进入 Step4。通知配置不重验工作流。

Step4：勾选要导出的 OCR 字段后点「保存」即可；允许 0 字段。有候选时须保存后才能公开；改勾选后须再保存。无候选时整页空态，可直接公开。保存不改场景「可发布/草稿」，也不重验工作流。

状态更新顺序：Step2 测通 → 场景可发布。Step3/Step4 始终可进入；Step4 保存只固化输出勾选。公开：可发布，且（已保存或无 OCR 候选）。从 Step3/Step4 退回 Step1/2：无实质改动则保持场景状态与测试结论；有改动则回草稿并作废测试。已发布后再改任一步并保存，编辑态回草稿，须重跑 Step2 测试后才能再公开。

#### 6.01.6 Step1～Step4 校验说明

Step1 提供关联检查；Step2 提供工作流测试，作为辅助检查，不强制阻断进入 Step3/Step4。Step3 没有测试，只保存通知配置；Step4 没有测试，只保存输出勾选。终了结构与到达在 Step2 测试内检查。公开时：须可发布；有 OCR 候选还须已保存。

步骤跳转与依赖总表：

| 从 | 到 | 跳转条件 | 依赖关系 |
| --- | --- | --- | --- |
| Step1 | Step2 | 当前 Step1 无阻断错误 | Step2 读取 Step1 的账票集合、主账票、关联关系和 OCR 字段 |
| Step2 | Step3 | 始终可进入 | Step2 的测试结论不阻断进入，但画布实质变更后测试结论会失效 |
| Step3 | Step4 | 点击「次へ」或已保存后继续 | Step3 保存的是通知配置；Step4 读取已保存的通知设置和 Step2 输出变量 |
| Step4 | 公開 | 场景可发布，且有 OCR 候选时 Step4 已保存 | 发布时读取 Step1/Step2/Step3/Step4 的最终保存态 |

状态依赖总表：

| 改动来源 | 影响 |
| --- | --- |
| Step1 改动 | 会影响 Step2 画布、Step3 通知变量可达性、Step4 输出候选；破坏性改动时阻断保存 |
| Step2 改动 | 会使 Step2 测试结论失效；若删除或改线导致 Step3/Step4 已引用变量失效，保存时阻断 |
| Step3 改动 | 仅影响通知配置本身；不重置 Step2 测试结论 |
| Step4 改动 | 仅影响输出勾选；不重置 Step2 测试结论 |


| 步骤    | 触发按钮/动作    | 检查对象                      | 成功后写入          | 失败后写入                | 是否影响场景可发布  |
| ----- | ---------- | ------------------------- | -------------- | -------------------- | ---------- |
| Step1 | 関連チェック     | 关联账票是否已连接主账票              | 关联检查通过         | 关联检查失败；提示未连接主账票的关联账票 | 间接（影响能否测通） |
| Step2 | テスト実行 / 次へ | 工作流测试（含结构、连线、结束到达、模拟执行等）  | 测试结果记录为通过/失败 | 测试失败仅提示；不阻断进入 Step3/Step4 | 否          |
| Step3 | 保存 / 次へ  | 通知配置可为空；有配置时校验通知对象、主题、内容字段格式 | 保存通知配置          | 字段格式错误时提示并要求修正       | 否          |
| Step4 | 保存         | 固化当前 OCR 账票/字段勾选（允许 0 字段） | 记为已保存          | 本期不因字段数量失败           | 否（只影响能否公开） |
| Step4 | 公開         | 场景可发布；有候选须已保存             | 场景状态置为已发布      | 提示先测通或先保存            | 是          |


测试输出要求：


| 步骤    | 输出项  | 内容                                              |
| ----- | ---- | ----------------------------------------------- |
| Step1 | 错误列表 | 未连接主账票的关联账票列表                                   |
| Step2 | 错误列表 | 层级、规则 ID、节点、字段、原因、处理建议                          |
| Step2 | 画布定位 | 点击错误项后高亮对应节点或连线                                 |
| Step2 | 时间线  | 展示模拟执行顺序、自动节点跳过原因、失败节点、人工确认待办上下文校验结果和测试跳过人工等待说明 |
| Step2 | 发布状态 | Step2 工作流测试成功时，将场景状态置为可发布；失败则保持草稿，但不阻断进入 Step3/Step4 |
| Step3 | 错误列表 | 通知对象、主题、内容格式错误时提示；通知配置为空时允许继续                     |
| Step4 | 错误列表 | 本期保存不做强校验阻断；联动导致候选消失时侧栏空态提示                     |


Step1 关联检查只检查一件事：是否存在没有和主账票连上的关联账票。账票数量、主账票、主字段、匹配策略在添加账票或字段选择时已经校验；关系字段有效性、关系方向、重复关系和后续引用保护不属于关联检查。


| 规则 ID | 检查项       | 通过条件                 | 失败条件 / 处理               |
| ----- | --------- | -------------------- | ----------------------- |
| S1-01 | 关联账票连接主账票 | 每个关联账票至少有 1 条关系连到主账票 | 任一关联账票未连到主账票时，提示未关联账票列表 |


Step2 工作流测试规则按维度执行。测试遇到阻断规则或任一节点失败时，立即停止当前轮测试；后续节点不再执行，时间线保持未执行。错误列表记录维度、规则 ID、节点/字段、原因和处理建议。

本期测试输入固定为产品内置「标准集约完成案件」快照（只读，不可替换）。因此不校验「内置用例是否存在」「用例是否有文件」——这两项是交付物前提，不是配置人员可触发的失败态。用例维度只保留与场景配置可能漂移的一项：样例账票是否仍在 Step1 集合中。


| 维度   | 规则 ID | 检查项        | 通过条件                                                                | 失败条件 / 处理                                 |
| ---- | ----- | ---------- | ------------------------------------------------------------------- | ----------------------------------------- |
| 用例   | S2-03 | 用例账票仍有效    | 用例中的主账票与关联账票类型都仍在 Step1 账票集合中（参考资料除外）                                  | 任一该类账票类型已从 Step1 删除时测试阻断                    |
| 结构   | S2-04 | 开始节点       | 画布存在且只存在 1 个开始节点；开始节点不可删除                                           | 缺失或重复开始节点时测试阻断                            |
| 结构   | S2-05 | 结束节点       | 至少存在 1 个结束节点                                                        | 缺失结束节点或结束节点后仍有连线时测试阻断                     |
| 结构   | S2-06 | 有效路径       | 从开始节点出发的每条有效分支都能到达结束节点                                              | 任一有效分支不可达结束节点时测试阻断并高亮断点                   |
| 结构   | S2-07 | 孤立节点       | 必经处理节点都在从开始到结束的可达路径上                                                | 存在孤立必经节点时测试阻断                             |
| 结构   | S2-08 | 回流路径       | 回流连线允许存在，但回流后仍必须能到达结束节点，且测试时间线标注为环状路径                               | 回流造成不可终止路径时测试阻断                           |
| 连线   | S2-09 | 主输入数量      | 每个非开始节点最多 1 条主输入                                                    | 同一节点存在多条主输入时测试阻断                          |
| 连线   | S2-10 | 条件出口       | 条件节点 IF / ELIF / ELSE 出口不重复，ELSE 最多 1 条                             | 出口重复或 ELSE 重复时测试阻断                        |
| 连线   | S2-11 | 人工确认出口     | 人工确认同一出口不重复连接；至少连接 1 个出口                                            | 同一出口多连或没有任何出口时测试阻断                        |
| 连线   | S2-12 | 连线端点       | 每条连线的起点和终点节点都存在                                                     | 存在悬空连线时测试阻断                               |

### 6.02 工作流画布与可编排节点



#### 6.02.1 概述

画布只编排**案件集约完成之后**的处理段；不编排上传、案件集约、导出。节点规则在账票类型、数据映射、AI 检证等设置页维护，画布配置外壳、开关、引用与分支。

- 各节点变量统一说明出力、消费路径、允许取值和可用运算符。
- 配置端「出力変数」不提供折叠按钮；列表行只显示裸变量键 + 类型；点圆形 i 查看中文变量名和取值范围。i 气泡标签用日文（変数名 / 取値範囲 / 例），不展示长说明；`files[]` 只附 1 条短 JSON 示例。所有节点都直接平铺展示，不再显示案件/文件/账票类型分组标题。
- 画布节点不再展示配置摘要行；节点卡片只显示类型名（条件/人工确认额外显示分支结构）。



#### 6.02.2 功能点

1. 画布编辑：添加/插入/连接/删除节点与连线、自动整列；逻辑变更置测试为未测试。节点标题固定为类型名。
2. 节点选择器：业务 Agent（前处理、OCR、映射、检证）、控制（条件、人工确认）、扩展（自定义函数）、端点（开始、结束）。
3. 节点编辑面板：按类型展示参数；底部「出力変数」不折叠，只读展示变量键 + 类型；中文变量名、取值范围、`files[]` 短示例在 i 中。映射、检证仅跳转设置页。
4. 变量引用：条件仅选可达上游变量；自定义函数固定读取开始节点 `files[]`。会破坏既有引用的保存阻断，发布前仍发现不可达引用时发布阻断。
5. 分支：条件 IF/ELIF/ELSE；人工确认三出口（完成/补件/案件终止），审查者选出口即路由。
6. 人工确认：须接在前处理/OCR/检证之后；只接需要的出口，至少一条。
7. 工作流测试：内置集约完成用例；不跑上传/集约，不写正式案件/待办。测试成功后场景置为可发布；测试不作为进入 Step3/Step4 的门禁（终了结构与到达在测试内完成，不另层校验）。
8. Step1 派生 schema：Step2 的账票类别和 OCR 字段只来自 Step1 关联账票集合；Step2 不能新增、删除或重命名账票类别和 OCR 字段。
9. 说明图标统一为圆形 i；画布不再提供「全部折叠节点」入口。复制场景后不展示 Step1「要確認」引用检查横幅。

#### 6.02.8 工作流测试

Step2 画布用工作流测试证明「当前画布配置可跑通」。测试只影响场景是否可发布，不阻断进入 Step3/Step4；它不是正式跑件：不创建正式案件、不写真实待办。规则维度按 S2-03～S2-33（用例维仅校验样例账票与 Step1 是否仍一致）；本节只补「怎么操作、样例长什么样、成功/失败写什么、弹什么提示」。

功能点：

1. 配置人员在 Step2 打开测试弹窗，确认内置样例后执行测试。
2. 系统按「用例 → 结构 → 连线 → 节点配置 → 变量引用 → 字段选择 → 依赖 → 模拟执行 → 发布门禁」顺序校验；前一层阻断则不进入后一层。结束节点有无、无出边、可达结束等归结构/模拟维度，不在测试外再挂一层。
3. 模拟执行按样例字段求值后的命中路径生成时间线（条件走真分支，人工确认跳过真人动作后按已连接出口继续）；任一节点失败立即停止，后续节点保持未执行。
4. 全部规则通过：写测试结论=成功，场景状态置为可发布。
5. 任一失败（含未配置结束、不可达结束）：写测试结论=失败，场景保持草稿（非已发布时），但不阻断进入 Step3/Step4。
6. 人工确认节点在测试中跳过真实确认动作（不等人、不建真实待办），校验上下文与出口后继续下游，以便测通后续节点。正式运行仍须真人确认。

#### 6.02.10 工作流变量

本节先按消费场景说明变量怎么被使用，再按节点说明配置、校验和输出变量。消费场景固定为：条件、通知、系统、待办。独立通知节点已删除；通知能力内置在 Step3 通知设置。

#### 出力変数展示规则（配置端，对齐画布右侧检视）

选中画布节点后，右侧检视面板底部展示「出力変数」。规则如下：


| 项 | 规则 |
| --- | --- |
| 出现条件 | 当前为节点检视，且该节点有至少 1 条输出变量；无输出时整块不展示 |
| 折叠 | 不提供折叠按钮；标题栏界面文案：出力変数；标题旁展示变量条数 |
| 分组 | 不显示分组标题；所有节点统一平铺 |
| 列表行 | 只显示「裸变量键 + 类型」；不显示说明、不显示消费路径列 |
| 说明入口 | 每行右侧圆形 i；悬停弹出气泡 |
| i 气泡内容 | 日文标签；只显示中文变量名、取值范围；`files[]` 只显示 1 条短 JSON 示例；不展示长说明 |

| 画布卡片 | 节点卡片不展示出力变量摘要行 |


出力変数面板列全量输出，不代表该变量可被条件直接选用；条件选择器另按消费路径过滤。

#### 变量粒度分类


| 粒度 | 定义 | 典型变量 | 是否可进条件变量池 | 说明 |
| --- | --- | --- | --- | --- |
| 案件级变量 | 一个案件实例只有一个值，或由多个文件/字段汇聚成一个案件级结论 | `preprocessStatus`、`preprocessResult`、`ocrStatus`、`ocrResult`、`mappingStatus`、`mappingResult`、`verifyStatus`、`verifyResult`、`hitlStatus`、`codeStatus` | 只有消费分类含条件、且来自上游可达节点时可选 | 条件节点主要消费处理状态与处理结果 |
| 文件级变量 | 仅开始节点以 `files[]` 数组承载归属文件信息 | `files[]` | 不可直接选 | 传给处理节点与人工确认页读取；条件不用数组 |
| 账票字段级变量 | 从 Step1 账票模板字段或标准字段中选到具体叶子字段后生成 | `docTypes.{账票类型}.{OCR字段名}`、`standardFields.{字段名}` | 当前版本按 String 类型可选 | 字段容器不可选；必须选到叶子字段 |


`files[]` 仅开始节点输出。人工确认页、日志和系统执行仍可读取运行时文件对象，但不作为其他节点的出力变量。

#### 处理状态与处理结果（业务处理节点共用）

前处理、OCR、数据映射、AI 检证统一拆成两个案件级枚举：


| 变量 | 取值 | 白话 |
| --- | --- | --- |
| 处理状态 | `processing` | 还在跑 |
| 处理状态 | `success` | 跑完了且没系统失败；没东西可处理时也记成功（旧所说的 skip） |
| 处理状态 | `failed` | 跑挂了（系统/配置/依赖异常） |
| 处理结果 | `passed` | 业务通过，可走主流程 |
| 处理结果 | `reviewRequired` | 业务要确认，去人工确认 |


分岔时：先看处理状态有没有挂（`failed`），再看处理结果要不要人看（`reviewRequired`）。

Step3 通知设置使用内置通知策略，不在页面展示条件表达式。页面只配置四个固定模块：前处理、OCR 抽出、数据映射、AI 检证。每行只保留左侧模块标题与变量名、右侧配置计数。页面只填写通知对象、主题、内容和可插入变量。通知对象支持多选。插入变量时先切换件名/内容，再从下拉选择变量，插入栏放在件名和内容上面。配置存在即启用；删除配置即不触发。点击保存或下一步时校验并保存通知配置；通知配置为空时允许继续。页面右上提供「リセット」，用于清空当前 Step3 通知配置。

内置触发条件：

| 通知类型 | 可配置节点 | 触发条件 | 说明 |
| --- | --- | --- | --- |
| 失败通知 | 前处理 | `{node}.case.preprocessStatus = failed` | 代表系统、依赖或配置异常 |
| 失败通知 | OCR 抽出 | `{node}.case.ocrStatus = failed` | 代表系统、依赖或配置异常 |
| 失败通知 | 数据映射 | `{node}.case.mappingStatus = failed` | 代表系统、依赖或配置异常 |
| 失败通知 | AI 检证 | `{node}.case.verifyStatus = failed` | 代表系统、依赖或配置异常 |
| 要确认通知 | 前处理 | `{node}.case.preprocessStatus = success AND {node}.case.preprocessResult = reviewRequired` | 代表节点正常跑完，但业务结果需要人工看 |
| 要确认通知 | OCR 抽出 | `{node}.case.ocrStatus = success AND {node}.case.ocrResult = reviewRequired` | 代表节点正常跑完，但业务结果需要人工看 |
| 要确认通知 | 数据映射 | `{node}.case.mappingStatus = success AND {node}.case.mappingResult = reviewRequired` | 代表节点正常跑完，但业务结果需要人工看 |
| 要确认通知 | AI 检证 | `{node}.case.verifyStatus = success AND {node}.case.verifyResult = reviewRequired` | 代表节点正常跑完，但业务结果需要人工看 |

人工确认节点和自定义函数节点没有 `result`，因此只支持失败通知。开始节点、结束节点、条件节点不配置通知。

自定义函数节点和人工确认节点只输出各自节点状态键：`hitlStatus` / `codeStatus`，取值同处理状态：`processing / success / failed`。它们不输出 `result`、`action` 或错误消息变量。

#### 变量消费分类

案件状态是平台固定聚合状态，不是 Step2 节点变量。条件节点不能直接选择案件状态。


| 消费分类 | 谁消费 | 可消费变量 | 不可消费变量 | 说明 |
| ---- | --- | --- | --- | --- |
| 条件 | 条件判断节点 | 上游可达、消费分类含条件的变量；标准字段和 Step1 OCR 字段必须选到叶子 | 开始节点输出、结束节点输出、`files[]`、Array 容器、Date、DateTime、系统变量 | 只用于 IF / ELIF 分岔 |
| 通知 | Step3 通知模板 | 可通知节点的状态/结果变量，以及页面允许插入的上游变量 | 触发条件本身不可配置 | 只用于主题和内容插值 |
| 系统 | 引擎、日志、状态汇聚、Step4 | 开始 `caseId` / `files[]`、映射 `standardFields` 等 | 除非变量表另行标注，不提供给条件 | 可在出力変数面板展示 |
| 待办 | 人工确认页、待办列表、审计详情 | 运行时文件与确认结果等 | 条件表达式直接比较数组 | 分岔使用状态/结果变量 |


同一业务事件的推荐写法：


| 场景 | 条件变量 | 待办/运行时 |
| --- | --- | --- |
| 结果要确认 | `{node}.case.*Result = reviewRequired` | 人工确认页展示明细 |
| 节点失败 | `{node}.case.*Status = failed` | 异常分支或终止 |
| 全部通过 | `*Status = success` 且 `*Result = passed` | 走主流程 |

| `codeStatus`       | Enum   | 条件   | `processing / success / failed` | Python 脚本处理状态；不输出 result 或错误消息变量 |




#### Step1 账票模板字段

Step1 账票模板字段不是节点输出变量。条件节点可以读取上游 OCR 结果聚合后的账票字段叶子，用于简单分岔。`帳票フィールド` 和 `{帳票タイプ}` 只是级联分组；只有选到第三级 `{OCR字段名}` 后，才按 String 类型展示对应运算子。

条件节点变量级联：


| 级   | 界面文案             | 含义                |
| --- | ---------------- | ----------------- |
| L1  | 帳票フィールド          | Step1 账票 OCR 字段分组 |
| L2  | {帳票タイプ}          | 具体账票类型名           |
| L3  | {OCR Field Name} | 该类型模板中的 OCR 字段    |


变量规则：


| 变量键                        | 类型     | 消费分类 | 可用运算子         | 定义                                |
| -------------------------- | ------ | ---- | ------------- | --------------------------------- |
| `docTypes.{账票类型}.{OCR字段名}` | String | 条件   | 按 String 类型展示 | 从 `files[].ocrFields` 聚合得到的运行时字段值 |

# vNeosAI 日本保险 IDP — 产品需求文档（PRD）

## 第 6 章｜功能详细说明

### 6.01 业务场景设置

业务场景设置包含三类场景：新建场景、编辑已发布场景、复制已有场景。三类场景共用三步配置和发布校验。

#### 功能点

1. 新建场景
  - 配置人员从场景列表创建空草稿。
  - 系统生成业务场景 ID，状态为草稿。
  - 配置人员必须完成账票集合、工作流画布、输出字段和工作流测试后才能发布。
2. 编辑已发布场景
  - 配置人员可直接打开已发布场景继续编辑。
  - 保存或修改工作流后，场景状态回到草稿，需重新测试并发布。
3. 复制已有场景
  - 系统复制基础信息、账票集合、主副账票关系、工作流定义和输出字段。
  - 复制后状态为待确认。
  - 配置人员必须复核被复制的账票、字段和规则引用；存在失效引用时阻断发布。
4. 账票关系配置
  - 配置业务场景名称、说明、账票集合、主账票、主字段、关联账票字段关系和匹配策略。
  - 账票类型候选来自账票类型设置。
  - 主账票必须且只能有一个；关联账票可有多个。
  - 每个关联账票至少配置一组字段关系。
5. 工作流入口
  - 配置人员从第 2 步进入工作流画布。
  - 工作流画布读取第 1 步账票集合和主副账票关系，限制节点可选账票、字段和规则范围。
  - 画布保存后回写最近测试结论、错误数量和可发布状态。
6. 输出字段
  - 配置人员选择对外输出字段和顺序。
  - 字段来源包括 OCR 抽出字段、标准字段（第 2 步配置数据映射节点时可用）、工作流结果字段。
  - 输出字段被导出和对外 API 消费。
7. 发布
  - 发布前校验三步配置、工作流测试、依赖引用和输出字段。
  - 发布成功后场景状态变为已发布，新案件使用当前配置。

### 6.02 工作流画布与可编排节点

工作流画布包含三类场景：编辑画布、配置节点、执行测试。画布只编排案件集约完成后的处理段，不编排上传、案件集约和导出。章节顺序：画布与节点配置 → 系统节点（开始/结束）→ 节点 I/O 与输出变量 → 工作流测试 → 联动与异常。

#### 功能点

1. 画布编辑
  - 配置人员从业务场景设置进入画布。
  - 支持添加节点、插入节点、连接节点、删除节点、删除连线、自动整理。
  - 任何影响执行逻辑的修改都会把工作流测试结论置为未测试。
2. 节点选择器
  - 业务 Agent：AI 前处理、OCR 抽出、数据映射、AI 检证。
  - 控制节点：条件判断、人工确认、通知。
  - 扩展节点：自定义函数。
  - 端点：开始、结束。
  - 通过画布右下角工具栏「ノードを追加」或节点连接点唤起浮层选择器。
3. 节点配置
  - 选中节点后，在右侧节点编辑面板配置参数；节点编辑面板标题旁 ⓘ 悬停展示节点说明，画布节点标题下展示摘要。
  - 处理节点底部以折叠区展示「出力変数」（输出变量）；条件判断、通知等通过变量级联选择器引用上游输出变量。
  - 数据映射、AI 检证节点不在节点编辑面板内编辑规则明细，仅提供跳转至对应设置页。
4. 上游变量引用
  - 条件判断、通知变量插入、自定义函数入参等，只可引用当前路径可达的上游输出变量。
5. 分支与出口
  - 普通节点默认一个完成出口。
  - 条件判断节点支持 IF、ELIF、ELSE。
  - 人工确认节点固定三种结果分支：完成、补件、案件终止。
6. 待办生成
  - AI 前处理节点可生成前处理确认待办。
  - OCR 抽出节点可生成 OCR 抽出确认待办。
  - AI 检证节点可生成 AI 检证确认待办。
  - 人工确认节点须接在前处理、OCR 或 AI 检证之后（如中间插入条件节点，自动穿透检测上游最近的业务agent节点）；上游不合法时测试标记失败。
7. 调用逻辑
  - 案件集约完成后，齐套新建案件时启动工作流实例；补件归入、补件直挂或后续文件归入时续跑已有实例；处理中止后重新执行时绑定当前生效工作流版本启动新实例。
  - 工作流按顺序执行每个节点，节点内按文件并行处理，处理完后进入下一个工作流节点。
  - AI 前处理和 OCR 抽出按文件执行，并向上汇聚节点进度。
  - 数据映射、AI 检证、条件判断、通知、自定义函数按案件执行。
  - 人工确认节点生成待办后暂停所在分支，待办办结后按动作出口继续。
  - 到达结束节点时当前分支结束；所有活跃分支均结束且无未办结人工确认待办时，工作流完结并触发导出待办。
8. 工作流测试
  - 测试对象是当前工作流草稿与按业务场景保存的集约済み测试用例；不执行上传与案件集约，不生成正式案件、正式待办。
  - 系统内置一组标准已集约完成的测试用例；用户可上传 ZIP/JSON 覆盖并持久化到当前业务场景，下次打开自动沿用；也可恢复内置默认。
  - 点击「执行测试」后，先校验测试用例必填，再沿从开始节点可达的路径依次模拟各节点；人工确认节点标记为跳过。
  - 若人工确认上游拓扑不合法，该节点标记失败并展示错误原因。
  - 全部节点通过且到达结束节点时为测试成功；否则为测试失败。
  - 测试成功后，业务场景才可发布。

#### 变量作用域与引用规则

工作流变量按作用域分为三类，与节点编辑面板输出变量折叠区、条件判断变量选择器、通知变量插入中的展示一致。


| 作用域   | 界面标注  | 含义                     | 典型引用路径                   |
| ----- | ----- | ---------------------- | ------------------------ |
| 案件级   | 案件    | 本案一次节点执行后的聚合结果，不区分单文件  | `{节点变量名.case.项目键}`       |
| 文件级   | ファイル  | 本案每个逻辑文件各一份，按文件 ID 索引  | `{节点变量名.files[].项目键}`    |
| 账票类型级 | 帳票タイプ | 与 Step1 关联账票类型绑定的清单或枚举 | `{节点变量名.docTypes[].项目键}` |


引用规则：

1. 节点变量名由引擎按节点在画布上的唯一标识生成；配置端只读展示，不可改。
2. 条件判断、通知变量插入、自定义函数入参仅可选择当前路径可达的上游节点输出变量。
3. 对象、数组类型变量在条件表达式中须选择子字段、件数或是否为空等派生项后再比较。
4. 上游节点删除、账票类型移除、映射规则或检证规则失效时，下游引用标记为错误并阻断发布。
5. 文件级变量在导出配置中须明确展开方式（按文件、按账票类型聚合等）。

界面文案：出力変数；引用格式说明——後続ノード・IF/ELSE 条件で `{ノード変数名.項目}` 形式で指定します。

#### 工作流测试流程

测试在测试弹窗内完成。案件集约不在测试范围内；测试输入为内置标准已集约完成的用例，或用户上传并已保存的 ZIP/JSON 快照。配置阶段无法从案件一览选取真实案件，因案件仅在业务场景发布并受理后才会产生。

执行顺序（用户只点击一次「执行测试」，系统内部按序执行）：

1. 测试用例校验：检查案件编号、案件名、文件清单；无效则提示并中止。
2. 节点模拟执行：沿从开始节点可达的路径依次模拟各节点；环路径节点在时间线标注；人工确认节点标记为跳过。
3. 结论汇总：无失败节点且到达结束节点时为测试成功；人工确认上游拓扑不合法时该节点失败；其余失败场景为测试失败。结论回写业务场景设置中的最近测试结论与可发布状态。

界面文案：测试弹窗标题——Workflow テスト；时间线标题——Workflow 実行；执行中按钮文案——実行中…；成功提示——Workflow テストを実行しました；上传成功——テストデータを更新しました：{文件名}；恢复默认——初期値に戻す。

#### 测试输入


| 输入方式   | 说明                  | 要求                                |
| ------ | ------------------- | --------------------------------- |
| 内置标准用例 | 系统预置的集约済み案件快照（只读展示） | 打开测试弹窗即加载；无已保存记录时使用此默认值           |
| 上传覆盖   | ZIP / JSON 集约済み快照文件 | 上传后自动覆盖当前用例并按业务场景持久化；仅支持 ZIP/JSON |
| 恢复默认   | 将当前用例重置为内置标准用例      | 仅在上传覆盖后显示「恢复默认」按钮                 |


说明：

- 不从案件一览选取真实案件；配置发生在受理之前，案件尚不存在。
- 不提供案件集约执行；测试用例描述的是已集约完成的案件输入前提。
- 文件清单中的文件名可包含 PDF、图片、TIFF 等账票扩展名；缺件、OCR 低置信、检证不通过等结果不在测试输入中预设，而在 Workflow 模拟执行时由对应节点（尤其 AI 检证节点）判定。
- 原型不解析真实 ZIP 内容；JSON 可解析完整元数据结构；模拟执行依据当前用例元数据（案件信息 + 文件清单）。

#### 测试校验

原型当前实现的校验分为测试执行校验与发布校验两类。

测试执行校验（点击「执行测试」时）


| 校验项    | 检查对象     | 阻断条件                 | 用户提示 / 处理                    |
| ------ | -------- | -------------------- | ---------------------------- |
| 测试用例   | 测试弹窗内用例  | 案件编号、案件名或文件清单不完整     | 「案件番号を入力してください」等对应提示         |
| 人工确认上游 | 人工确认节点拓扑 | 未接在前处理、OCR 或 AI 检证之后 | 时间线该节点标记失败，展示须接在可产生待办节点之后的说明 |


发布校验（业务场景发布时）


| 校验项   | 检查对象  | 阻断条件          | 用户提示 / 处理               |
| ----- | ----- | ------------- | ----------------------- |
| 账票配置  | 第 1 步 | 未添加关联账票或关联不完整 | 对应 Step1 校验提示           |
| 工作流节点 | 画布    | 无任何节点         | 「Workflowノードを設定してください」  |
| 输出字段  | 第 3 步 | 未选择导出字段       | 「エクスポート字段を1件以上選択してください」 |


测试结论映射：


| 测试结果           | 工作流测试结论 | 是否可发布 |
| -------------- | ------- | ----- |
| 存在失败节点或未到达结束节点 | 测试失败    | 否     |
| 全部通过且到达结束节点    | 测试成功    | 是     |


说明：原型不在测试弹窗内执行完整的画布结构校验（如开始/结束节点数量、分支完整性、变量引用合法性等）；环路径节点允许存在，时间线标注「環状パス上のノード」。

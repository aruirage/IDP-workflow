### 6.01 业务场景设置

#### 6.01.1 概述

三步向导：Step1 案件集约规则，Step2 跳转工作流画布，Step3 输出字段与发布。新建、编辑已发布、复制三类场景共用流程；影响执行逻辑的变更将 Step2 工作流测试结论置为未测试，影响输出的变更将 Step3 输出测试结论置为未测试。

#### 6.01.2 功能点

1. 场景列表：新建、复制、重命名、删除、关键词检索；复制后状态为待确认，须执行跨步骤引用检查。
2. Step1：配置名称、说明、账票集合（1～20）、主账票与主字段、关联账票字段关系、匹配策略；支持关联检查与 AI 自动关联。
3. Step2：进入工作流画布；画布读取 Step1 账票与关系，限制节点可选范围。
4. Step3：勾选并排序输出字段（OCR 字段、标准字段、工作流结果字段等）。
5. 发布：三步校验通过、Step2 工作流测试成功且 Step3 输出测试成功后方可发布；新案件使用发布后配置。

状态更新顺序：Step2 的测试按钮只执行工作流画布测试，写 Step2 工作流测试结论。Step3 不提供单独保存按钮；点击保存并测试时，系统先保存 Step3 并把状态置为待确认，只检查输出字段、导出格式、输出目标，写 Step3 输出测试结论。只有 Step2 工作流测试成功且 Step3 输出测试成功时，场景状态才改为可发布。点击发布时只接受可发布状态，发布成功后改为已发布。已发布场景再次修改任一步骤并保存时，不直接覆盖已发布版本，当前编辑态回到待确认，必须重新测试、重新发布。

#### 6.01.6 Step1～Step3 测试项目与结论

Step1、Step2、Step3 的测试职责分开。Step1 只检查是否存在未和主账票连上的关联账票；Step2 检查工作流画布是否能按内置用例跑通；Step3 检查输出配置是否可用。Step3 不打开 Step2 的工作流测试弹窗，不重新模拟画布执行，也不重新判断结束节点字段的业务取值。


| 步骤  | 触发按钮/动作                 | 测试对象             | 成功后写入                     | 失败后写入                     | 是否影响场景可发布 |
| --- | ----------------------- | ---------------- | ------------------------- | ------------------------- | -------- |
| Step1 | 関連チェック              | 关联账票是否已连接主账票 | 关联检查通过 | 关联检查失败；提示未连接主账票的关联账票 | 是        |
| Step2 | テスト / テスト実行          | 工作流画布结构、节点配置、变量引用、依赖、模拟执行 | Step2 工作流测试结论 = 测试成功       | Step2 工作流测试结论 = 测试失败       | 是        |
| Step3 | 保存してテスト              | 输出字段、显示名、排序、导出格式、输出目标 | Step3 输出测试结论 = 测试成功；若 Step2 已成功则场景状态置为可发布 | Step3 输出测试结论 = 测试失败；场景状态保持待确认 | 是        |


Step1 关联检查只检查一件事：是否存在没有和主账票连上的关联账票。账票数量、主账票、主字段、匹配策略在添加账票或字段选择时已经校验；关系字段有效性、关系方向、重复关系和后续引用保护不属于关联检查。


| 规则 ID | 检查项      | 通过条件                         | 失败条件 / 处理              |
| ----- | -------- | ---------------------------- | ---------------------- |
| S1-01 | 关联账票连接主账票 | 每个关联账票至少有 1 条关系连到主账票        | 任一关联账票未连到主账票时，提示未关联账票列表 |


Step2 工作流测试规则按维度执行。测试遇到阻断规则立即停止当前轮测试，并在错误列表中记录维度、规则 ID、节点/字段、原因和处理建议。


| 维度   | 规则 ID | 检查项       | 通过条件                                                   | 失败条件 / 处理                                      |
| ---- | ----- | --------- | ------------------------------------------------------ | ---------------------------------------------- |
| 用例   | S2-01 | 内置用例存在    | 系统能加载内置集约完成用例                                         | 用例缺失时测试阻断                                      |
| 用例   | S2-02 | 用例文件有效    | 用例至少包含 1 个已归属逻辑文件                                     | 没有文件或文件未归属时测试阻断                                |

| 结构   | S2-04 | 开始节点       | 画布存在且只存在 1 个开始节点；开始节点不可删除                           | 缺失或重复开始节点时测试阻断                                  |
| 结构   | S2-05 | 结束节点       | 至少存在 1 个结束节点；结束节点没有出边                                | 缺失结束节点或结束节点后仍有连线时测试阻断                          |

### 6.02 工作流画布与可编排节点

#### 6.02.1 概述

画布只编排**案件集约完成之后**的处理段；不编排上传、案件集约、导出。节点规则在账票类型、数据映射、AI 检证等设置页维护，画布配置外壳、开关、引用与分支。

- 各节点变量统一说明出力、消费路径、允许取值和可用运算符。
- 配置端「出力変数」折叠区：变量键 + 类型 + 界面文案；不展示取值说明。文件级统一仅列 `files[]` 一项。

#### 6.02.2 功能点

1. 画布编辑：添加/插入/连接/删除节点与连线、自动整列；逻辑变更置测试为未测试。节点标题固定为类型名。
2. 节点选择器：业务 Agent（前处理、OCR、映射、检证）、控制（条件、人工确认、通知）、扩展（自定义函数）、端点（开始、结束）。
3. 节点编辑面板：按类型展示参数；底部「出力変数」只读。映射、检证仅跳转设置页。
4. 变量引用：条件、通知、函数入参仅选可达上游变量；会破坏既有引用的保存阻断，发布前仍发现不可达引用时发布阻断。
5. 分支：条件 IF/ELIF/ELSE；人工确认三出口（完成/补件/案件终止），审查者选出口即路由。
6. 人工确认：须接在前处理/OCR/检证之后；只接需要的出口，至少一条。
7. 工作流测试：内置集约済み用例；不跑上传/集约，不写正式案件/待办。
8. Step1 派生 schema：Step2 的账票类别和 OCR 字段只来自 Step1 关联账票集合；Step2 不能新增、删除或重命名账票类别和 OCR 字段。

#### 6.02.8 工作流测试

**6.02.8.1 输入与执行**：内置集约済み用例，只读；点「テスト実行」→ 校验 → 模拟执行 → 扫描结论。测试 runner 遇到人工确认节点时不等待真人处理，不创建真实待办，也不模拟审查者提交；只校验该节点能否生成待办上下文和出口路由，然后按已连接出口继续扫描。这里的测试跳过只表示“跳过人工等待”，不写 `confirmStatus=skipped`，也不代表运行时画布可以无确认内容跳过人工确认节点。

**6.02.8.2 校验层级**：按固定顺序执行：用例 → 结构 → 节点配置 → 变量引用 → 依赖 → 模拟执行 → 发布门禁。前一层有阻断时不继续执行后一层；全部层级通过时写 Step2 工作流测试成功，任一层级不通过时写 Step2 工作流测试失败。


| 层级   | 具体校验项                                                                                                                 | 不通过处理            |
| ---- | --------------------------------------------------------------------------------------------------------------------- | ---------------- |
| 用例   | 必须存在内置集约完成用例；用例至少包含一个已归属逻辑文件；用例账票类型必须仍在 Step1 配置中                                                                     | 测试阻断             |
| 结构   | 必须存在开始节点和结束节点；开始节点不可删除；结束节点不可有出边；每条有效分支必须可达结束节点；不可存在孤立必经节点                                                            | 测试阻断，画布高亮错误节点或连线 |
| 连线   | 节点输入端最多一条主输入；条件节点 IF / ELIF / ELSE 出口不可重复；人工确认同一出口不可重复连接                                                              | 测试阻断             |
| 节点配置 | 前处理至少开启一项能力；OCR 至少开启一个账票；映射和 AI 检证引用的设置必须存在且已发布；人工确认必须配置审查角色且至少连接一个出口；通知必须配置渠道、收件人、标题、正文；自定义函数脚本不能为空                  | 测试阻断             |
| 变量引用 | 条件左值必须来自上游可达条件变量；通知插入变量必须来自上游可达通知变量；自定义函数入参必须来自上游可达变量；删除节点、改线或配置变更导致引用不可达时必须报错                                        | 测试阻断             |
| 字段选择 | 标准字段和 OCR 字段必须选到叶子字段；`standardFields`、`docTypes[]`、`files[]`、Array、Date、DateTime、结束节点输出不可作为条件左值                       | 测试阻断             |
| 依赖   | 数据映射运行前必须已有前处理/OCR 结果；AI 检证运行前必须已有文件、OCR、映射依赖；人工确认只开放给前处理、OCR 抽出、AI 检证；若人工确认前一跳是条件节点，测试时必须穿透条件节点，找到最近的非条件上游处理节点并校验其类型 | 测试阻断或节点等待        |
| 模拟执行 | 按画布顺序模拟节点状态；人工确认不创建真实待办，只检查待办上下文是否可生成；通知不发送真实消息，只校验模板变量可替换                                                            | 失败节点标红，测试失败      |
| 发布门禁 | Step2 工作流测试必须成功；Step1/Step2 任一影响执行的配置变更后，Step2 工作流测试结论回到未测试。Step3 输出测试在业务场景设置中单独执行，不属于本弹窗                             | 发布阻断             |

#### 6.02.10 工作流变量

本节先按消费场景说明变量怎么被使用，再按节点说明配置、校验和输出变量。消费场景固定为四类：条件、通知、系统、待办。不存在混合消费类别；同一变量如果同时可用于待办和通知，在变量表里分别标注“待办 / 通知”。

画布摘要通则：节点标题下方单行展示；段间以 `·` 分隔；无有效配置时显示 `未設定`（数据映射无已配规则时显示 `設定参照`）；开始节点和结束节点不显示摘要。结构上首段为计数或状态摘要（如 `{n}/{total} 有効`、`{n}件有効`、`ルール {n}件`），后续段为明细项，明细最多再展示 2 项。

#### 变量消费分类


| 消费分类 | 谁消费                   | 可消费变量                                        | 不可消费变量                                                         | 说明                                      |
| ---- | --------------------- | -------------------------------------------- | -------------------------------------------------------------- | --------------------------------------- |
| 条件   | 条件判断节点                | 上游可达、案件级、消费分类含条件的变量；标准字段和 Step1 OCR 字段必须选到叶子 | 开始节点输出、结束节点输出、`files[]`、Array、Date、DateTime、系统变量、待办明细、通知正文专用变量 | 只用于 IF / ELIF 分岔。补件、再执行、案件状态触发不在条件节点里判断 |
| 通知   | 通知节点标题和正文             | 上游可达、消费分类含通知的案件级变量；开始节点 `caseNo` 可插入         | 结束节点输出、Date、DateTime、`files[]`、待办结构化明细                         | 结束前要通知，必须把通知节点放在结束节点之前                  |
| 系统   | 引擎、日志、状态汇聚、Step3 输出字段 | 开始上下文、分支命中结果、结束结果、执行时间、文件数组、账票类型数组           | 不提供给条件节点或通知正文，除非变量表另行标注                                        | 系统变量可在出力変数面板展示，但不代表能被 Step2 节点消费        |
| 待办   | 人工确认页、待办列表、审计详情       | `files[]`、缺失清单、冲突清单、AI 检证逐条规则结果、人工修正摘要       | 条件表达式、通知模板直接插入                                                 | 待办展示明细；分岔要使用对应状态变量，不直接比较数组              |


同一业务事件的推荐写法：


| 场景     | 条件变量                                               | 待办变量                      | 通知变量                       |
| ------ | -------------------------------------------------- | ------------------------- | -------------------------- |
| 必备账票缺失 | `requiredDocumentStatus = reviewRequired`          | `missingDocuments` 展示缺失账票 | `missingDocuments` 插入补件通知  |
| 必填字段缺失 | `requiredFieldStatus = reviewRequired`             | `missingFields` 展示缺失字段    | `missingFields` 插入补正通知     |
| 映射冲突   | `mappingConflictStatus = reviewRequired`           | `mappingConflicts` 展示冲突来源 | `mappingConflicts` 可插入异常通知 |
| 节点执行异常 | `preprocessStatus = failed` 或 `ocrStatus = failed` | `files[]` 展示关联文件          | `errorMessage` 插入异常通知      |


#### 条件变量规则

条件节点只能选择上游连线可达变量。开始节点和结束节点不是条件变量来源；开始节点负责启动实例，结束节点是 Step2 终点。条件节点变量选择器不得出现 Array、Date、DateTime、`files[]`、字段容器 `docTypes[]`、字段容器 `standardFields`、`aiVerifyResultJson`、`missingDocuments`、`missingFields`、`mappingConflicts`、`mappingErrors`。

条件运算子统一使用英文：


| 类型              | 可用运算子                                                                 | 默认     | 说明                                        |
| --------------- | --------------------------------------------------------------------- | ------ | ----------------------------------------- |
| Enum            | `=` / `≠`                                                             | `=`    | 状态和枚举字段                                   |
| Boolean         | `true` / `false`                                                      | `true` | 右值为布尔选项，不显示额外输入框                          |
| Number          | `=` / `≠` / `>` / `≥` / `<` / `≤`                                     | `=`    | 仅节点内置 Number 输出；标准字段和 OCR 字段不用 Number 运算子 |
| String          | `=` / `≠` / `contains` / `not contains` / `is empty` / `is not empty` | `=`    | 文本类字段                                     |
| Array           | 不可选                                                                   | —      | 用对应状态变量分岔                                 |
| Date / DateTime | 不可选                                                                   | —      | 不出现在条件节点                                  |


字段容器选择规则：

`docTypes[]` 和 `standardFields` 都是字段容器，不是可比较变量。条件节点只在用户选到具体子字段后才生成可比较变量；当前版本标准字段和 OCR 字段叶子统一按 String 处理。

开发实现规则：变量选择器前两级只用于分组和检索，不允许保存为条件左值。用户必须选到最后一级叶子字段，系统才生成条件表达式左值，并按 String 类型加载运算子和右值控件。未选到叶子字段时，「保存」「发布」均按条件未完成处理。

级联选择与变量生成规则：


| 字段来源            | 选择层级                               | 生成的条件左值                    | 类型     | 运行时取值                              |
| --------------- | ---------------------------------- | -------------------------- | ------ | ---------------------------------- |
| Step1 账票 OCR 字段 | `帳票フィールド` → `{帳票タイプ}` → `{OCR字段名}` | `docTypes.{账票类型}.{OCR字段名}` | String | 从上游 `files[].ocrFields` 聚合到当前案件    |
| 数据映射标准字段        | `{映射节点}` → `標準フィールド` → `{标准字段名}`   | `standardFields.{标准字段名}`   | String | 从数据映射节点 `standardFields` 中按标准字段名取值 |
| 字段组容器           | `docTypes[]`、`standardFields`      | 不生成条件左值                    | —      | 仅用于级联分组展示或系统传递                     |


标准字段和 OCR 字段的运行时规则：当前版本统一按 String 进入条件变量选择器。金额、日期、枚举等业务含义仍可由字段名和上游映射/检证规则表达，但条件节点不根据这些业务含义切换 Number、Date、Boolean 或 Enum 运算子。

四态状态通用定义：


| 值                | 定义                                        |
| ---------------- | ----------------------------------------- |
| `passed`         | 节点或检证类型已执行，结果满足规则，无需人工介入                  |
| `reviewRequired` | 已执行，但存在低置信、缺失、冲突、不满足规则，或 AI 无法给出通过结论      |
| `failed`         | 执行异常、依赖异常、配置错误或系统错误，无法产出可信结果              |
| `skipped`        | 未执行，原因是开关关闭、无适用输入、无适用规则或运行时判断无需执行，且按非阻断处理 |


条件变量 catalog：


| 来源           | 变量键                        | 类型      | 取值范围                                         | 取值定义                                                                                       | 典型条件场景                                                                         |

| AI 检证节点      | `requiredFieldStatus`      | Enum    | `passed / reviewRequired / failed / skipped` | `passed`=适用必填规则已执行且字段都有值；`reviewRequired`=必填字段为空或无法证明存在；`failed`=检证异常；`skipped`=模块关闭或无适用规则 | `requiredFieldStatus = reviewRequired` → 字段补正                                  |

#### 通知变量规则

| 来源     | 变量键                | 类型       | 取值范围                                                        | 消费方        | 定义                 |

| 开始节点   | `docTypes[]`       | Array    | Step1 账票类型列表                                                | 系统         | 运行时匹配账票模板；字段容器不作条件 |

#### 结束节点

节点定位：结束节点是 Step2 画布终点。结束节点不能有出边，不能再接条件、通知、人工确认或处理节点。结束节点输出只供系统、执行日志、案件状态汇聚和 Step3 输出字段消费，不进入 Step2 条件变量池，也不进入 Step2 通知变量池。

| --------- | -------- | -------------- |

| 区块标题      | 終了時提案    | 只读展示终局结果说明     |
| 出力変数（折叠区） | 出力変数     | 展示 Step3 可消费变量 |


校验规则：


| 校验项            | 白话规则                                                                        | 不通过处理       |
| -------------- | --------------------------------------------------------------------------- | ----------- |
| 必须有结束节点        | 每条有效分支最终都要能走到一个结束节点                                                         | 发布阻断        |
| 结束节点不能有出边      | 结束就是终点，后面不能再接任何节点                                                           | 保存或发布阻断     |
| 所有分支必须可终了      | IF / ELIF / ELSE、人工确认出口只要存在连线，就必须能到结束节点                                     | 发布阻断        |

| `missingDocuments`       | Array   | 待办 / 通知 | 账票类型名数组                                      | 缺失账票列表     |
| `missingFields`          | Array   | 待办 / 通知 | 字段名数组                                        | 缺失字段列表     |
| `aiVerifyResultJson`     | Array   | 待办      | 元素含 `ruleId / verificationType / result`     | 已执行规则结果列表  |
| `files[]`                | Array   | 待办 / 系统 | 含 `verificationResults` 的文件数组                | 文件关联检证结果   |


#### 人工确认节点

节点定位：人工确认节点固定三出口：完成、补件、案件终止。审查者选择动作后路由，不需要再写条件表达式。只接需要的出口，至少一条；未接出口不出现在待办。当前版本人工确认只开放给前处理、OCR 抽出、AI 检证。中间允许经过条件节点，但测试和发布校验必须穿透条件节点，确认最近的非条件上游处理节点属于这三类。人工确认节点不是自动处理节点，不使用 `skipped`；如果没有确认内容，应由上游条件分支避开该节点。

配置界面（日语）：


| 区块/控件  | 界面文案（日语）       | 说明                             |
| ------ | -------------- | ------------------------------ |
| 面板标题   | 人工確認           | 右侧检视面板顶栏                       |
| 区块标题   | ゲート設定          | 审查角色配置                         |
| 只读字段   | 確認対象           | 确认内容由上游自动判定                    |
| 下拉字段   | 審査ロール          | 选项：案件担当者 / 給付審査 / 管理者 / その他ロール |
| 画布出口标签 | 完成 / 補件 / 案件終止 | 画布节点体与连线标签                     |


校验规则：


| 校验项      | 规则                                                      |
| -------- | ------------------------------------------------------- |
| 审查角色必填   | 未配置角色保存阻断                                               |
| 至少连接一个出口 | 无出口发布阻断                                                 |
| 出口语义固定   | 完成写 `approve`，补件写 `requestSupplement`，案件终止写 `stopCase`  |
| 未接出口不可触发 | 审查页不展示未接出口                                              |
| 上游类型限制   | 最近的非条件上游处理节点必须是前处理、OCR 抽出或 AI 检证；否则发布阻断                 |
| 条件节点穿透   | 如果人工确认直接接在条件节点后，沿该条件分支反向穿透连续条件节点，找到最近的非条件上游处理节点再做上游类型校验 |


出口连接校验：


| 出口   | 可连接下游                 | 校验规则                                                       |
| ---- | --------------------- | ---------------------------------------------------------- |
| 完成   | 后续处理节点、通知节点、条件节点、结束节点 | 连接后运行时写 `confirmAction=approve`；必须能继续走到结束节点                |
| 补件   | 通知节点或结束节点             | 连接后运行时写 `confirmAction=requestSupplement`；分支最终只能进入补件待处理或结束 |
| 案件终止 | 通知节点或结束节点             | 连接后运行时写 `confirmAction=stopCase`；分支最终只能进入中止结束              |


运行时校验：


| 校验项      | 规则                                                                                                                  |
| -------- | ------------------------------------------------------------------------------------------------------------------- |
| 确认内容必须存在 | 到达人工确认节点时必须存在确认内容；上游状态为 `reviewRequired`、`failed` 或存在待办明细时才允许生成待办；没有确认内容时按配置或运行时错误处理，不创建待办，不写 `confirmStatus`，流程不继续 |
| 待办创建     | 到达人工确认节点时写 `confirmStatus=open`；只展示已连接出口对应的动作按钮                                                                     |
| 完成动作     | 审查者选择完成后写 `confirmStatus=completed`、`confirmAction=approve`；未连完成出口时不可提交                                             |
| 补件动作     | 审查者选择补件后写 `confirmStatus=completed`、`confirmAction=requestSupplement`；未连补件出口时不可提交                                   |
| 案件终止动作   | 审查者选择案件终止后写 `confirmStatus=completed`、`confirmAction=stopCase`；未连案件终止出口时不可提交                                        |
| 写回失败     | 待办提交、变量写回或路由失败时写 `confirmStatus=failed`，不继续走下游                                                                      |
| 重复提交     | 已 `completed` 或 `failed` 的待办不可再次提交；重复请求返回幂等结果                                                                       |
| 权限校验     | 只有配置的审查角色、案件担当者或管理员可处理待办；无权限提交返回失败                                                                                  |

| --- | ------------------ | ---------------- |
| 已配置 | `{確認対象} · {審査ロール}` | `OCR結果確認 · 給付審査` |
| 缺角色 | `{確認対象} · 案件担当者`   | `AI検証確認 · 案件担当者` |

| --------------- | ----- | ------- | ---------------------------------------- | ---------- |
| `confirmStatus` | Enum  | 条件      | `open / completed / failed`              | 人工确认待办生命周期 |
| `confirmAction` | Enum  | 条件      | `approve / requestSupplement / stopCase` | 审查者选择的出口   |

#### 条件判断节点

节点定位：IF / ELIF 每行一条比较式；同分支多条件以 AND / OR 连接，默认 AND。ELSE 无表达式。多值匹配通过多行条件加 OR 表达，不提供单独的 in 运算子。

配置界面（日语）：


| 区块/控件 | 界面文案（日语）    | 说明                   |
| ----- | ----------- | -------------------- |
| 面板标题  | IF/ELSE     | 右侧检视面板顶栏             |
| 区块标题  | 分岐条件        | IF / ELIF / ELSE 编辑区 |
| 逻辑连接  | AND / OR    | 多条件时切换               |
| 变量选择  | 変数を選択       | 只显示条件变量 catalog      |
| 运算子   | 按左值类型列出     | 运算子统一使用英文            |
| 比较值   | Enter value | 需右值时展示               |


校验规则：


| 校验项             | 规则                                          |
| --------------- | ------------------------------------------- |
| IF / ELIF 必须有条件 | 空条件发布阻断                                     |
| ELSE 不允许条件      | ELSE 只作为兜底分支                                |
| 变量必须上游可达        | 节点删除、连线变化或配置变更导致变量不可达时保存阻断；发布前仍发现不可达引用时发布阻断 |
| 左值必须是条件变量       | Array、Date、DateTime、开始节点输出、结束节点输出不可选        |
| 运算子必须匹配类型       | 类型不匹配时保存阻断                                  |


典型条件：


| 场景                   | 条件式                                                     |
| -------------------- | ------------------------------------------------------- |
| OCR 低置信进入人工确认        | `{ocr}.case.lowConfidenceFieldCount > 0`                |
| 必备账票缺失进入补件           | `{verify}.case.requiredDocumentStatus = reviewRequired` |
| AI 检证全部通过走主流程        | `{verify}.case.verifyStatus = passed`                   |
| 映射冲突进入人工确认           | `{map}.case.mappingConflictStatus = reviewRequired`     |
| 标准字段包含关键字            | `{map}.case.standardFields.policyNo contains "A"`       |
| Step1 账票 OCR 字段包含关键字 | `docTypes.invoice.claimNo contains "A"`                 |


输出变量：


| 变量键                | 类型      | 消费分类 | 取值范围                 | 定义              |
| ------------------ | ------- | ---- | -------------------- | --------------- |
| `branchName`       | String  | 系统   | IF / ELIF / ELSE 分支名 | 本次命中的分支名        |
| `branchResult`     | Boolean | 系统   | `true / false`       | 条件是否成立          |
| `matchedFileCount` | Number  | 系统   | 非负整数                 | 文件级条件命中数；当前不进条件 |

### 6.01 业务场景设置



#### 6.01.1 概述

三步向导：Step1 案件集约规则，Step2 跳转工作流画布，Step3 输出字段与发布。新建、编辑已发布、复制三类场景共用流程。进入 Step3 要求 Step2 工作流测试通过，且画布状态已转为可发布。

#### 6.01.2 功能点

1. 场景列表：新建、复制、重命名、删除、关键词检索；复制后进入 Step1 继续编辑。
2. Step1：配置名称、说明、账票集合（1～20）、主账票与主字段、关联账票字段关系、匹配策略；支持关联检查与 AI 自动关联。
3. Step2：进入工作流画布；画布读取 Step1 账票与关系，限制节点可选范围。工具栏提供工作流测试；测试成功后，画布状态置为可发布，方可进入 Step3。
4. Step3：从 OCR 抽出账票类型中勾选要导出的字段（允许 0）。进入须先满足 Step2 门禁。点保存固化勾选，不重跑画布测试。
5. 发布：场景须为可发布；有 OCR 候选时还须已保存输出勾选；无候选时可直接公开。新案件只读取已发布版本。

场景状态触发条件：


| 显示状态 | 实现取值        | 触发条件                                                                                      | 按钮影响                                                                       |
| ---- | ----------- | ----------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| 草稿   | `draft`     | 新建/复制后；或 Step2 未测试/测试失败；或从 Step3 回到 Step1/2 后修改了配置；或修改画布后测试作废；旧 `pending_review` 读档时回退为草稿 | 允许保存；不允许发布；不可进入 Step3                                                      |
| 可发布  | `ready`     | Step2 工作流测试成功（终了结构、可达结束等已含在测试内）                                                           | 允许进入 Step3；正式公开还须（已保存输出勾选，或无 OCR 候选）；从 Step3 退回 Step1/2 若无改动则保持可发布；有改动则回草稿 |
| 已发布  | `published` | 用户在可发布且（已保存输出，或无 OCR 候选）时点击发布并通过确认                                                        | 发布按钮置灰；新案件读取该版本；从 Step3 退回 Step1/2 若无改动则保持已发布；有改动则回草稿                      |


Step3：勾选要导出的 OCR 字段后点「保存」即可；允许 0 字段。有候选时须保存后才能公开；改勾选后须再保存。无候选时整页空态，可直接公开。保存不改场景「可发布/草稿」，也不重验工作流。

状态更新顺序：Step2 测通 → 场景可发布 → 可进 Step3。Step3 保存只固化输出勾选。公开：可发布，且（已保存或无 OCR 候选）。从 Step3 退回 Step1/2：无实质改动则保持场景状态与测试结论；有改动则回草稿并作废测试。已发布后再改任一步并保存，编辑态回草稿，须重跑 Step2 测试后才能再公开。

#### 6.01.6 Step1～Step3 校验说明

Step1 提供关联检查；Step2 提供工作流测试，并作为进入 Step3 与场景「可发布」的门禁；Step3 没有测试，只保存输出勾选。终了结构与到达在 Step2 测试内检查。公开时：须可发布；有 OCR 候选还须已保存。


| 步骤    | 触发按钮/动作    | 检查对象                      | 成功后写入          | 失败后写入                | 是否影响场景可发布  |
| ----- | ---------- | ------------------------- | -------------- | -------------------- | ---------- |
| Step1 | 関連チェック     | 关联账票是否已连接主账票              | 关联检查通过         | 关联检查失败；提示未连接主账票的关联账票 | 间接（影响能否测通） |
| Step2 | テスト実行 / 次へ | 工作流测试（含结构、连线、结束到达、模拟执行等）  | 测试成功；场景状态置为可发布 | 测试失败；保持草稿；阻断进入 Step3 | 是          |
| Step3 | 保存         | 固化当前 OCR 账票/字段勾选（允许 0 字段） | 记为已保存          | 本期不因字段数量失败           | 否（只影响能否公开） |
| Step3 | 公開         | 场景可发布；有候选须已保存             | 场景状态置为已发布      | 提示先测通或先保存            | 是          |


测试输出要求：


| 步骤    | 输出项  | 内容                                              |
| ----- | ---- | ----------------------------------------------- |
| Step1 | 错误列表 | 未连接主账票的关联账票列表                                   |
| Step2 | 错误列表 | 层级、规则 ID、节点、字段、原因、处理建议                          |
| Step2 | 画布定位 | 点击错误项后高亮对应节点或连线                                 |
| Step2 | 时间线  | 展示模拟执行顺序、自动节点跳过原因、失败节点、人工确认待办上下文校验结果和测试跳过人工等待说明 |
| Step2 | 发布状态 | Step2 工作流测试成功时，将场景状态置为可发布；失败则保持草稿并阻断进入 Step3    |
| Step3 | 错误列表 | 本期保存不做强校验阻断；联动导致候选消失时侧栏空态提示                     |


Step1 关联检查只检查一件事：是否存在没有和主账票连上的关联账票。账票数量、主账票、主字段、匹配策略在添加账票或字段选择时已经校验；关系字段有效性、关系方向、重复关系和后续引用保护不属于关联检查。


| 规则 ID | 检查项       | 通过条件                 | 失败条件 / 处理               |
| ----- | --------- | -------------------- | ----------------------- |
| S1-01 | 关联账票连接主账票 | 每个关联账票至少有 1 条关系连到主账票 | 任一关联账票未连到主账票时，提示未关联账票列表 |


Step2 工作流测试规则按维度执行。测试遇到阻断规则或任一节点失败时，立即停止当前轮测试；后续节点不再执行，时间线保持未执行。错误列表记录维度、规则 ID、节点/字段、原因和处理建议。


| 维度   | 规则 ID | 检查项        | 通过条件                                                                | 失败条件 / 处理                                 |
| ---- | ----- | ---------- | ------------------------------------------------------------------- | ----------------------------------------- |
| 用例   | S2-01 | 内置用例存在     | 系统能加载内置集约完成用例                                                       | 用例缺失时测试阻断                                 |
| 用例   | S2-02 | 用例文件有效     | 用例至少包含 1 个已归属逻辑文件                                                   | 没有文件或文件未归属时测试阻断                           |
| 用例   | S2-03 | 用例账票仍有效    | 用例中的账票类型都仍在 Step1 账票集合中                                             | 任一账票类型已从 Step1 删除时测试阻断                    |
| 结构   | S2-04 | 开始节点       | 画布存在且只存在 1 个开始节点；开始节点不可删除                                           | 缺失或重复开始节点时测试阻断                            |
| 结构   | S2-05 | 结束节点       | 至少存在 1 个结束节点                                                        | 缺失结束节点或结束节点后仍有连线时测试阻断                     |

### 6.02 工作流画布与可编排节点



#### 6.02.1 概述

画布只编排**案件集约完成之后**的处理段；不编排上传、案件集约、导出。节点规则在账票类型、数据映射、AI 检证等设置页维护，画布配置外壳、开关、引用与分支。

- 各节点变量统一说明出力、消费路径、允许取值和可用运算符。
- 配置端「出力変数」可折叠、默认展开，切换节点后重新展开；列表行只显示裸变量键 + 类型；点圆形 i 查看名称、取值范围与说明；Array（含 `files[]`）在 i 中附带 JSON 示例。文件级统一仅列 `files[]` 一项。分组标题界面文案：案件変数 / ファイル変数 / 帳票タイプ変数。
- 画布节点不再展示配置摘要行；节点卡片只显示类型名（条件/人工确认额外显示分支结构）。



#### 6.02.2 功能点

1. 画布编辑：添加/插入/连接/删除节点与连线、自动整列；逻辑变更置测试为未测试。节点标题固定为类型名。
2. 节点选择器：业务 Agent（前处理、OCR、映射、检证）、控制（条件、人工确认、通知）、扩展（自定义函数）、端点（开始、结束）。
3. 节点编辑面板：按类型展示参数；底部「出力変数」可折叠、默认展开只读列表（变量键 + 类型；说明/取值范围/Array 示例在 i 中）。映射、检证仅跳转设置页。
4. 变量引用：条件、通知、函数入参仅选可达上游变量；会破坏既有引用的保存阻断，发布前仍发现不可达引用时发布阻断。
5. 分支：条件 IF/ELIF/ELSE；人工确认三出口（完成/补件/案件终止），审查者选出口即路由。
6. 人工确认：须接在前处理/OCR/检证之后；只接需要的出口，至少一条。
7. 工作流测试：内置集约完成用例；不跑上传/集约，不写正式案件/待办。测试成功后场景置为可发布，作为进入 Step3 的门禁（终了结构与到达在测试内完成，不另层校验）。
8. Step1 派生 schema：Step2 的账票类别和 OCR 字段只来自 Step1 关联账票集合；Step2 不能新增、删除或重命名账票类别和 OCR 字段。
9. 说明图标统一为圆形 i；画布不再提供「全部折叠节点」入口。复制场景后不展示 Step1「要確認」引用检查横幅。

#### 6.02.8 工作流测试

Step2 画布用工作流测试证明「当前画布配置可跑通」。测试是进入 Step3 与场景「可发布」的门禁，不是正式跑件：不创建正式案件、不写真实待办、不发真实通知。规则维度仍按 S2-01～S2-33；本节只补「怎么操作、样例长什么样、成功/失败写什么、弹什么提示」。

功能点：

1. 配置人员在 Step2 打开测试弹窗，确认内置样例后执行测试。
2. 系统按「用例 → 结构 → 连线 → 节点配置 → 变量引用 → 字段选择 → 依赖 → 模拟执行 → 发布门禁」顺序校验；前一层阻断则不进入后一层。结束节点有无、无出边、可达结束等归结构/模拟维度，不在测试外再挂一层。
3. 模拟执行按可达路径生成时间线；任一节点失败立即停止，后续节点保持未执行。
4. 全部规则通过：写测试结论=成功，场景状态置为可发布，允许进入 Step3。
5. 任一失败（含未配置结束、不可达结束）：写测试结论=失败，场景保持草稿（非已发布时），阻断进入 Step3。
6. 人工确认节点在测试中跳过真实确认动作（不等人、不建真实待办），校验上下文与出口后继续下游，以便测通后续节点。正式运行仍须真人确认。

#### 6.02.10 工作流变量

本节先按消费场景说明变量怎么被使用，再按节点说明配置、校验和输出变量。消费场景固定为：条件、通知、系统、待办；同一变量若同时服务待办与通知，在变量表里标注「待办 / 通知」（与配置端消费路径标签一致）。

#### 出力変数展示规则（配置端，对齐画布右侧检视）

选中画布节点后，右侧检视面板底部展示「出力変数」。规则如下：


| 项        | 规则                                                                                             |
| -------- | ---------------------------------------------------------------------------------------------- |
| 出现条件     | 当前为节点检视，且该节点有至少 1 条输出变量；无输出时整块不展示                                                              |
| 折叠       | 可折叠；标题栏界面文案：出力変数；标题旁展示变量条数                                                                     |
| 默认态      | 默认展开；切换选中节点时强制重新展开（不保留上一节点的收起态）                                                                |
| 分组       | 按粒度分组；组标题界面文案：案件変数 / ファイル変数 / 帳票タイプ変数（有则显示，无则省略该组）                                             |
| 列表行      | 只显示「裸变量键 + 类型」：裸键为完整变量 ID 的最后一段（如 `case.preprocessStatus` → `preprocessStatus`）；不显示说明、不显示消费路径列 |
| 说明入口     | 每行右侧圆形 i；悬停弹出气泡                                                                                |
| i 气泡内容   | 依次：变量中文/日语名称、取值范围（界面前缀「取值:」+ 允许取值说明）、说明文案；三者去重后换行拼接                                            |
| Array 示例 | 类型为 Array（含 `files[]`、`missingDocuments`、`aiVerifyResultJson` 等）时，在上述内容后追加「例:」与 JSON 示例        |
| 文件级      | 面板只列一项 `files[]`，不展开 `files[].xxx` 元素字段；元素 schema 仅见变量规格，不进面板行                                 |
| 画布卡片     | 节点卡片不展示出力变量摘要行                                                                                 |


出力変数面板列全量输出（含仅系统/待办消费的变量），不代表该变量可被条件或通知直接选用；条件/通知选择器另按消费路径过滤。

#### 变量粒度分类

变量先按数据粒度分三类，再按消费场景分条件、通知、系统、待办。粒度回答“变量代表什么数据”，消费分类回答“哪个节点或页面能用它”。


| 粒度      | 定义                                     | 典型变量                                                                              | 是否可进条件变量池                           | 说明                             |
| ------- | -------------------------------------- | --------------------------------------------------------------------------------- | ----------------------------------- | ------------------------------ |
| 案件级变量   | 一个案件实例只有一个值，或由多个文件/字段汇聚成一个案件级结论        | `preprocessStatus`、`ocrStatus`、`verifyStatus`、`confirmAction`                     | 只有消费分类含条件、且来自上游可达节点时可选              | 条件节点主要消费这一类状态变量；案件状态本身不属于可配置变量 |
| 文件级变量   | 一个案件下有多个逻辑文件，变量以 `files[]` 数组承载每个文件的信息 | `files[]`、`files[].ocrFields`、`files[].verificationResults`、`files[].manualEdits` | 不可直接选 `files[]` 或数组；只能通过状态变量或字段叶子判断 | 用于系统传递、待办展示、节点处理输入；不作为条件左值     |
| 账票字段级变量 | 从 Step1 账票模板字段、OCR 字段或标准字段中选到具体叶子字段后生成 | `docTypes.{账票类型}.{OCR字段名}`、`standardFields.{字段名}`                                 | 当前版本按 String 类型可选                   | 字段容器不可选；必须选到叶子字段，才生成可比较变量      |


`files[]` 是文件级变量的统一数组。节点面板的出力変数列表只展示 `files[]`，不展开每个文件字段；i 气泡展示说明、取值范围与 JSON 示例。人工确认页、日志和系统执行可以读取元素内的基础信息。


| 元素键                    | 类型       | 定义                                          |
| ---------------------- | -------- | ------------------------------------------- |
| `id`                   | String   | 逻辑文件唯一标识                                    |
| `name`                 | String   | 原始文件名                                       |
| `type`                 | String   | 文件 MIME 或扩展名种类                              |
| `extension`            | String   | 带点扩展名                                       |
| `url`                  | String   | 本节点执行完成时点可访问的存储 URL                         |
| `size`                 | Number   | 字节大小                                        |
| `caseId`               | String   | 所属案件 ID                                     |
| `classificationResult` | String   | 账票类型分类标签                                    |
| `status`               | Enum     | `pending / processing / processed / failed` |
| `uploadedAt`           | DateTime | 文件首次上传时间                                    |
| `updatedAt`            | DateTime | 本节点处理完成后的更新时间                               |
| `ocrFields`            | Object   | OCR 抽出字段键值对象；不进入条件或通知变量池，条件只能选择 OCR 字段叶子    |
| `verificationResults`  | Array    | 文件关联的 AI 检证结果                               |

| 消费分类 | 谁消费                   | 可消费变量                                                                                                           | 不可消费变量                                                         | 说明                                      |
| ---- | --------------------- | --------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------- | --------------------------------------- |
| 条件   | 条件判断节点                | 上游可达、案件级、消费分类含条件的变量；标准字段和 Step1 OCR 字段必须选到叶子                                                                    | 开始节点输出、结束节点输出、`files[]`、Array、Date、DateTime、系统变量、待办明细、通知正文专用变量 | 只用于 IF / ELIF 分岔。补件、再执行、案件状态触发不在条件节点里判断 |
| 通知   | 通知节点标题和正文             | 上游可达、消费分类含「通知」或「待办/通知」的案件级变量（如 `caseNo`、`errorMessage`、`missingDocuments`）；不展开条件专用状态、不展开 Step1 OCR 字段树、不展开标准字段树 | 结束节点输出、Date、DateTime、`files[]`、条件专用状态、任意 OCR/标准字段叶子            | 结束前要通知，必须把通知节点放在结束节点之前；插入变量保持精简         |
| 系统   | 引擎、日志、状态汇聚、Step3 输出字段 | 开始上下文、分支命中结果、结束结果、执行时间、文件数组、账票类型数组                                                                              | 不提供给条件节点或通知正文，除非变量表另行标注                                        | 系统变量可在出力変数面板展示，但不代表能被 Step2 节点消费        |
| 待办   | 人工确认页、待办列表、审计详情       | `files[]`、缺失清单、冲突清单、AI 检证逐条规则结果、人工修正摘要                                                                          | 条件表达式、通知模板直接插入                                                 | 待办展示明细；分岔要使用对应状态变量，不直接比较数组              |


同一业务事件的推荐写法：


| 场景     | 条件变量                                               | 待办变量                      | 通知变量                       |
| ------ | -------------------------------------------------- | ------------------------- | -------------------------- |
| 必备账票缺失 | `verifyStatus = reviewRequired`                    | `missingDocuments` 展示缺失账票 | `missingDocuments` 插入补件通知  |
| 必填字段缺失 | `verifyStatus = reviewRequired`                    | `missingFields` 展示缺失字段    | `missingFields` 插入补正通知     |
| 映射冲突   | `mappingConflictStatus = reviewRequired`           | `mappingConflicts` 展示冲突来源 | `mappingConflicts` 可插入异常通知 |
| 节点执行异常 | `preprocessStatus = failed` 或 `ocrStatus = failed` | `files[]` 展示关联文件          | `errorMessage` 插入异常通知      |




#### 条件变量规则

条件节点只能选择上游连线可达变量。开始节点和结束节点不是条件变量来源；开始节点负责启动实例，结束节点是 Step2 终点。条件节点变量选择器不得出现 Array、Date、DateTime、`files[]`、字段容器 `docTypes[]`、字段容器 `standardFields`、`aiVerifyResultJson`、`missingDocuments`、`missingFields`、`mappingConflicts`、`mappingErrors`。

条件运算子统一使用英文：


| 类型              | 可用运算子                                                                 | 默认     | 说明                                        |
| --------------- | --------------------------------------------------------------------- | ------ | ----------------------------------------- |
| Enum            | `=` / `≠`                                                             | `=`    | 状态和枚举字段                                   |
| Boolean         | `true` / `false`                                                      | `true` | 右值为布尔选项，不显示额外输入框                          |
| Number          | `=` / `≠` / `>` / `≥` / `<` / `≤`                                     | `=`    | 仅节点内置 Number 输出；标准字段和 OCR 字段不用 Number 运算子 |
| String          | `=` / `≠` / `contains` / `not contains` / `is empty` / `is not empty` | `=`    | 文本类字段                                     |
| Array           | 不可选                                                                   | —      | 用对应状态变量分岔                                 |
| Date / DateTime | 不可选                                                                   | —      | 不出现在条件节点                                  |


字段容器选择规则：

`docTypes[]` 和 `standardFields` 都是不可直接比较的聚合结构（`docTypes[]` 为 Array，`standardFields` 为 Object）。条件节点只在用户选到具体子字段后才生成可比较变量；当前版本标准字段和 OCR 字段叶子统一按 String 处理。

开发实现规则：变量选择器前两级只用于分组和检索，不允许保存为条件左值。用户必须选到最后一级叶子字段，系统才生成条件表达式左值，并按 String 类型加载运算子和右值控件。未选到叶子字段时，「保存」「发布」均按条件未完成处理。一级节点分组须带节点标识（来自节点 ID 的 `varName`）；二级叶子展示变量键原文，不用日文说明文案替换。

级联选择与变量生成规则：


| 字段来源            | 选择层级                               | 生成的条件左值                    | 类型     | 运行时取值                              |
| --------------- | ---------------------------------- | -------------------------- | ------ | ---------------------------------- |
| Step1 账票 OCR 字段 | `帳票フィールド` → `{帳票タイプ}` → `{OCR字段名}` | `docTypes.{账票类型}.{OCR字段名}` | String | 从上游 `files[].ocrFields` 聚合到当前案件    |
| 数据映射标准字段        | `{映射节点}` → `標準フィールド` → `{标准字段名}`   | `standardFields.{标准字段名}`   | String | 从数据映射节点 `standardFields` 中按标准字段名取值 |
| 字段组容器           | `docTypes[]`、`standardFields`      | 不生成条件左值                    | —      | 仅用于级联分组展示或系统传递                     |


标准字段和 OCR 字段的运行时规则：当前版本统一按 String 进入条件变量选择器。金额、日期、枚举等业务含义仍可由字段名和上游映射/检证规则表达，但条件节点不根据这些业务含义切换 Number、Date、Boolean 或 Enum 运算子。

四态状态通用定义：


| 值                | 定义                                        |
| ---------------- | ----------------------------------------- |
| `passed`         | 节点或检证类型已执行，结果满足规则，无需人工介入                  |
| `reviewRequired` | 已执行，但存在低置信、缺失、冲突、不满足规则，或 AI 无法给出通过结论      |
| `failed`         | 执行异常、依赖异常、配置错误或系统错误，无法产出可信结果              |
| `skipped`        | 未执行，原因是开关关闭、无适用输入、无适用规则或运行时判断无需执行，且按非阻断处理 |


条件变量 catalog：


| 来源           | 变量键                        | 类型     | 取值范围                                         | 取值定义                                                                                           | 典型条件场景                                                                         |
| ------------ | -------------------------- | ------ | -------------------------------------------- | ---------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| AI 前处理节点     | `preprocessStatus`         | Enum   | `passed / reviewRequired / failed / skipped` | `passed`=已执行且无需人工；`reviewRequired`=前处理结果需要人工确认；`failed`=执行异常；`skipped`=无适用文件或配置关闭              | `preprocessStatus = reviewRequired` → 前处理确认；`preprocessStatus = failed` → 异常通知 |
| OCR 抽出节点     | `ocrStatus`                | Enum   | `passed / reviewRequired / failed / skipped` | `passed`=OCR 可用；`reviewRequired`=低置信或分类不确定；`failed`=OCR 失败；`skipped`=无适用账票或开关关闭                | `ocrStatus = reviewRequired` → OCR 确认                                          |
| OCR 抽出节点     | `lowConfidenceFieldCount`  | Number | 非负整数                                         | 低置信 OCR 字段数量                                                                                   | `lowConfidenceFieldCount > 0` → OCR 确认                                         |
| 数据映射节点       | `mappingStatus`            | Enum   | `passed / reviewRequired / failed / skipped` | `passed`=映射完成；`reviewRequired`=存在需要人工判断的映射问题；`failed`=映射规则执行失败；`skipped`=无适用映射                 | `mappingStatus = failed` → 异常分支                                                |
| 数据映射节点       | `mappingConflictStatus`    | Enum   | `passed / reviewRequired / failed / skipped` | `passed`=无冲突；`reviewRequired`=多来源字段值冲突；`failed`=冲突检测异常；`skipped`=无可比对来源                        | `mappingConflictStatus = reviewRequired` → 人工确认                                |
| 数据映射节点       | `standardFields.{field}`   | String | 存储值为字符串；条件按 String 类型展示运算子                   | 标准字段叶子值；字段容器不可选                                                                                | `standardFields.policyNo contains "A"` → 保单号关键字分支                              |
| AI 检证节点      | `verifyStatus`             | Enum   | `passed / reviewRequired / failed / skipped` | `passed`=至少一条适用规则已执行且全部通过；`reviewRequired`=任一已执行规则要确认或缺件等；`failed`=节点级异常；`skipped`=无开启模块或无适用规则 | `verifyStatus = reviewRequired` → AI 检证确认或补件路由；`verifyStatus = failed` → 异常处理  |
| 人工确认节点       | `confirmStatus`            | Enum   | `open / completed / failed`                  | `open`=待办未处理；`completed`=审查者已提交；`failed`=提交或写回失败                                               | `confirmStatus = completed` → 继续流程                                             |
| 人工确认节点       | `confirmAction`            | Enum   | `approve / requestSupplement / stopCase`     | `approve`=通过；`requestSupplement`=要求补件；`stopCase`=终止案件                                          | `confirmAction = requestSupplement` → 补件结束                                     |
| 通知节点         | `notifySendStatus`         | Enum   | `sent / failed / skipped`                    | `sent`=发送成功；`failed`=发送失败；`skipped`=节点未执行或发送条件不满足                                              | `notifySendStatus = failed` → 备用通知或异常分支                                        |
| 自定义函数节点      | `status`                   | Enum   | `passed / reviewRequired / failed / skipped` | `passed`=脚本执行且业务通过；`reviewRequired`=脚本返回需人工判断；`failed`=脚本异常；`skipped`=无输入或配置跳过                 | `status = failed` → 异常分支                                                       |
| Step1 账票模板字段 | `docTypes.{账票类型}.{OCR字段名}` | String | 按 String 类型展示运算子                             | 从上游 OCR 文件字段聚合得到；非节点输出                                                                         | `docTypes.invoice.claimNo contains "A"`                                        |




#### 通知变量规则

通知节点只能插入上游可达、消费分类含通知的变量。不把条件专用变量、Step1 OCR 字段树、标准字段树放进通知插入候选。结束节点输出不进入通知变量池。典型可插：`caseNo`、`caseStatus`、`finalResult`、`errorMessage` / `lastFailureReason`、`missingDocuments`、`missingFields`。Date 和 DateTime 不可插入通知正文。

条件/通知变量选择器展示约定：

1. 一级显示节点日文类型名 + 节点标识（`varName`，由节点 ID 派生），用于区分同类型多个节点。
2. 二级显示输出变量键（如 `ocrStatus`），保持英文字面，不翻译成日文界面文案。
3. 运行时引用路径为 `{节点标识}.{变量键}`，依赖节点 ID 保证唯一，不靠类型名区分。


| 来源       | 变量键                   | 类型     | 取值范围                                         | 插入场景        |
| -------- | --------------------- | ------ | -------------------------------------------- | ----------- |
| 开始节点     | `caseNo`              | String | 案件编号字符串                                      | 通知标题或正文标识案件 |
| AI 前处理节点 | `errorMessage`        | String | 失败时非空字符串                                     | 前处理失败通知     |
| OCR 抽出节点 | `errorMessage`        | String | 失败时非空字符串                                     | OCR 失败通知    |
| 数据映射节点   | `mappingStatus`       | Enum   | `passed / reviewRequired / failed / skipped` | 通知映射结论      |
| 数据映射节点   | `mappingConflicts`    | Array  | 冲突明细数组                                       | 异常通知列出冲突    |
| 数据映射节点   | `mappingErrors`       | Array  | 错误明细数组                                       | 异常通知列出映射错误  |
| AI 检证节点  | `verifyStatus`        | Enum   | `passed / reviewRequired / failed / skipped` | 通知检证结论      |
| AI 检证节点  | `missingDocuments`    | Array  | 账票类型名数组                                      | 补件通知列出缺失账票  |
| AI 检证节点  | `missingFields`       | Array  | 字段名数组                                        | 补正通知列出缺失字段  |
| 通知节点     | `notifyFailureReason` | String | 失败时非空字符串                                     | 备用通知说明失败原因  |
| 自定义函数节点  | `errorMessage`        | String | 失败时非空字符串                                     | 函数异常通知      |


通知示例：


| 场景     | 模板变量                                                    |
| ------ | ------------------------------------------------------- |
| 补件通知   | `{{caseNo}}`、`{{missingDocuments}}`、`{{missingFields}}` |
| 异常通知   | `{{caseNo}}`、`{{verifyStatus}}`、`{{errorMessage}}`      |
| 映射冲突通知 | `{{caseNo}}`、`{{mappingConflicts}}`                     |




#### 系统变量规则

系统变量用于引擎执行、日志、状态汇聚和 Step3 输出字段，不进入 Step2 条件变量池。开始节点输出是系统上下文；结束节点输出只供系统和 Step3 消费。


| 来源     | 变量键                | 类型       | 取值范围                                                        | 消费方        | 定义                 |
| ------ | ------------------ | -------- | ----------------------------------------------------------- | ---------- | ------------------ |
| 开始节点   | `caseId`           | String   | 案件 ID                                                       | 系统         | 工作流实例绑定案件          |
| 开始节点   | `businessScene`    | String   | 场景名称或 ID                                                    | 系统         | 当前业务场景上下文          |
| 开始节点   | `docTypes[]`       | Array    | Step1 账票类型列表                                                | 系统         | 运行时匹配账票模板；字段容器不作条件 |
| 开始节点   | `files[]`          | Array    | 已归属文件数组                                                     | 系统 / 待办    | 传给处理节点和人工确认页       |
| 条件判断节点 | `branchName`       | String   | IF / ELIF / ELSE 分支名                                        | 系统         | 执行日志记录命中分支         |
| 条件判断节点 | `branchResult`     | Boolean  | `true / false`                                              | 系统         | 分支表达式是否成立          |
| 条件判断节点 | `matchedFileCount` | Number   | 非负整数                                                        | 系统         | 文件级条件命中数；当前不进条件    |
| 通知节点   | `notifiedAt`       | DateTime | ISO 8601                                                    | 系统         | 通知发送时间，不进通知正文      |
| 结束节点   | `finalResult`      | Enum     | `completed / supplementPending / exception / stopped`       | 系统 / Step3 | 本分支终局结果            |
| 结束节点   | `hasOpenItems`     | Boolean  | `true / false`                                              | 系统 / Step3 | 结束时是否仍有未完了事项       |
| 结束节点   | `resultFileStatus` | Enum     | `notApplicable / waiting / generating / completed / failed` | 系统 / Step3 | 成果文件生成状态           |
| 结束节点   | `endedAt`          | DateTime | ISO 8601                                                    | 系统 / Step3 | 到达结束节点的时间          |


结束节点输出不供 Step2 后续节点使用，因为结束节点不能有出边。要按结果通知，必须在结束节点之前用上游状态变量判断后接通知节点。

#### 待办变量规则

待办变量用于人工确认页展示，不直接作为条件左值。待办明细与条件状态要分开：条件看状态，待办看明细。


| 来源      | 变量键                   | 类型    | 取值范围      | 待办展示场景                   |
| ------- | --------------------- | ----- | --------- | ------------------------ |
| 处理节点    | `files[]`             | Array | 文件数组      | 展示原图、预览、OCR 字段、检证结果、人工修正 |
| 数据映射节点  | `mappingConflicts`    | Array | 冲突明细数组    | 显示哪几个来源字段值不一致            |
| 数据映射节点  | `mappingErrors`       | Array | 错误明细数组    | 显示映射规则失败原因               |
| AI 检证节点 | `missingDocuments`    | Array | 账票类型名数组   | 显示缺失账票                   |
| AI 检证节点 | `missingFields`       | Array | 字段名数组     | 显示缺失字段                   |
| AI 检证节点 | `aiVerifyResultJson`  | Array | 已执行规则结果数组 | AI 检证确认页逐条展示规则结果         |
| 人工确认节点  | `files[].manualEdits` | Array | 人工修正摘要数组  | 后续审计展示人工修改了什么            |


`aiVerifyResultJson` 元素结构：


| 元素键                | 类型     | 取值范围                                                                                                   | 定义                    |
| ------------------ | ------ | ------------------------------------------------------------------------------------------------------ | --------------------- |
| `ruleId`           | String | 规则标识                                                                                                   | AI 检证规则 ID            |
| `verificationType` | Enum   | `requiredField / requiredDocument / textValidation / dataValidation / mappingConflict / signatureSeal` | 检证类型                  |
| `result`           | Enum   | `passed / reviewRequired`                                                                              | 单条规则结果；未执行或不适用规则不生成元素 |




#### 开始节点

节点定位：开始节点是系统固定入口。它决定“这个案件实例能不能启动”，不决定画布分岔。开始节点不可删除，不可配置触发条件，不提供 Step2 条件变量。案件状态、补件续跑、再执行由引擎和操作入口判断，不作为开始节点输出变量。

配置界面（日语）：


| 区块/控件          | 界面文案（日语） | 说明                           |
| -------------- | -------- | ---------------------------- |
| 面板标题           | 開始       | 右侧检视面板顶栏                     |
| 区块标题           | 起動イベント   | 只读展示启动说明                     |
| 出力変数（可折叠，默认展开） | 出力変数     | 展示系统上下文变量；不表示可作条件；说明与取值范围在 i |


校验规则：


| 校验项          | 白话规则                          | 不通过处理        |
| ------------ | ----------------------------- | ------------ |
| 案件已完成集约      | 文件已经归到具体案件，才可以启动工作流           | 不启动实例        |
| 场景已发布        | 当前案件关联的业务场景必须是已发布版本           | 不启动实例，记录配置错误 |
| Step1 账票关系有效 | 本案文件能匹配到 Step1 配置里的账票类型和主账票关系 | 不启动实例或进入集约异常 |
| 至少有可处理文件     | 本案必须有归属成功且未废弃的逻辑文件            | 不启动实例        |
| 补件续跑由引擎判断    | 补件文件归属后，由案件和文件归属关系判断是否续跑      | 不生成条件变量      |
| 再执行由入口动作判断   | 再执行是操作端动作或系统动作，不是画布条件表达式      | 启动新实例或拒绝操作   |
| 节点结构固定       | 开始节点不可删除，且必须能到达至少一个结束节点       | 保存或发布阻断      |


输出变量（配置端可折叠、默认展开；行内仅变量键+类型；说明/取值范围/Array 示例在 i）：


| 变量键             | 类型     | 消费分类    | 条件可选 | 通知可插入 | 定义              |
| --------------- | ------ | ------- | ---- | ----- | --------------- |
| `caseId`        | String | 系统      | 否    | 否     | 案件唯一标识          |
| `caseNo`        | String | 通知      | 否    | 是     | 面向用户的案件编号       |
| `businessScene` | String | 系统      | 否    | 否     | 当前业务场景          |
| `docTypes[]`    | Array  | 系统      | 否    | 否     | Step1 已注册账票类型清单 |
| `files[]`       | Array  | 系统 / 待办 | 否    | 否     | 触发时已归属文件数组      |




#### 结束节点

节点定位：结束节点是 Step2 画布终点。结束节点不能有出边，不能再接条件、通知、人工确认或处理节点。结束节点输出只供系统、执行日志、案件状态汇聚和 Step3 输出字段消费，不进入 Step2 条件变量池，也不进入 Step2 通知变量池。

配置界面（日语）：


| 区块/控件          | 界面文案（日语） | 说明                        |

#### 通知节点

节点定位：发送系统通知或邮件。通知节点可以放在结束节点之前；不能放在结束节点之后。通知节点不改变案件状态。

配置界面（日语）：


| 区块/控件 | 界面文案（日语）      | 说明              |
| ----- | ------------- | --------------- |
| 面板标题  | 通知            | 右侧检视面板顶栏        |
| 区块标题  | 通知設定          | 渠道与收件人          |
| 下拉字段  | 通知方式          | 选项：システム通知 / メール |
| 收件人   | 通知先 / メールアドレス | 系统通知选角色；邮件输入地址  |
| 区块标题  | メッセージ内容       | 标题与正文           |
| 变量插入  | 変数を挿入         | 只显示通知变量池        |


校验规则：


| 校验项        | 规则                               |
| ---------- | -------------------------------- |
| 通知方式必填     | 未选择保存阻断                          |
| 收件人必填      | 无收件人保存阻断                         |
| 标题和正文必填    | 空内容保存阻断                          |
| 插入变量必须上游可达 | 保存时发现不可达引用则保存阻断；发布前仍发现不可达引用则发布阻断 |
| 结束节点变量不可插入 | 因为结束节点后不能再接通知                    |


输出变量（配置端可折叠、默认展开；行内仅变量键+类型；说明/取值范围/Array 示例在 i）：


| 变量键                   | 类型       | 消费分类 | 取值范围                      | 定义     |
| --------------------- | -------- | ---- | ------------------------- | ------ |
| `notifySendStatus`    | Enum     | 条件   | `sent / failed / skipped` | 通知发送结果 |
| `notifiedAt`          | DateTime | 系统   | ISO 8601                  | 通知发送时刻 |
| `notifyFailureReason` | String   | 通知   | 失败时非空字符串                  | 发送失败原因 |




#### 自定义函数节点

节点定位：用户定义入参和 Python 脚本。脚本可返回调试用结果，但用户自定义出参不进入条件、通知、系统、待办变量池。下游节点只能消费本节点内置输出：`status` 和 `errorMessage`。

入参规则：

1. 数据类型可选：`string` / `int` / `dict` / `array` / `float`。
2. 来源（ソース）默认是上游节点全部 `files[]` 型输出汇总成的 JSON；可选改为某一上游节点的 `files[]`。
3. 入参添加完成后，才可在脚本区「変数を挿入」中按参数名插入；脚本内用 `inputs['参数名']` 引用。

配置界面（日语）：


| 区块/控件 | 界面文案（日语）     | 说明                                        |
| ----- | ------------ | ----------------------------------------- |
| 面板标题  | カスタム関数       | 右侧检视面板顶栏                                  |
| 区块标题  | 入力変数         | 入参列表；追加时选参数名、数据类型、来源                      |
| 字段    | データ型         | 可选 string / int / dict / array / float    |
| 字段    | ソース          | 默认「上流ノードの files[] JSON」；可改选单个上游 `files[]` |
| 区块标题  | Python スクリプト | 脚本编辑区；「変数を挿入」只列出已添加入参名                    |
| 区块标题  | 実行結果         | 仅展示脚本返回值预览和执行日志；不生成下游变量                   |


校验规则：


| 校验项        | 规则                                         |
| ---------- | ------------------------------------------ |
| 脚本必填       | 空脚本保存阻断                                    |
| 入参来源有效     | 默认上游 files[] JSON；若改选单个 `files[]`，该变量须上游可达 |
| 脚本插入限制     | 「変数を挿入」仅在存在已添加入参时可选用，插入内容为 `inputs['参数名']` |
| 脚本返回值不生成变量 | 返回值仅用于本节点日志和调试预览，不进入任何变量池                  |
| 条件可选限制     | 条件节点只展示本节点内置 `status`，不展示脚本返回字段            |
| 通知可插入限制    | 通知节点只可插入本节点内置 `errorMessage`，不展示脚本返回字段     |


输出变量（配置端可折叠、默认展开；行内仅变量键+类型；说明/取值范围/Array 示例在 i）：


| 变量键            | 类型     | 消费分类 | 取值范围                                         | 定义          |
| -------------- | ------ | ---- | -------------------------------------------- | ----------- |
| `status`       | Enum   | 条件   | `passed / reviewRequired / failed / skipped` | 脚本执行和业务判断状态 |
| `errorMessage` | String | 通知   | 失败时非空字符串                                     | 脚本失败详情      |




#### Step1 账票模板字段

Step1 账票模板字段不是节点输出变量。条件节点可以读取上游 OCR 结果聚合后的账票字段叶子，用于简单分岔。`帳票フィールド` 和 `{帳票タイプ}` 只是级联分组；只有选到第三级 `{OCR字段名}` 后，才按 String 类型展示对应运算子。

条件节点变量级联：


| 级   | 界面文案             | 含义                |
| --- | ---------------- | ----------------- |
| L1  | 帳票フィールド          | Step1 账票 OCR 字段分组 |
| L2  | {帳票タイプ}          | 具体账票类型名           |
| L3  | {OCR Field Name} | 该类型模板中的 OCR 字段    |


变量规则：


| 变量键                        | 类型     | 消费分类 | 可用运算子         | 定义                                |
| -------------------------- | ------ | ---- | ------------- | --------------------------------- |
| `docTypes.{账票类型}.{OCR字段名}` | String | 条件   | 按 String 类型展示 | 从 `files[].ocrFields` 聚合得到的运行时字段值 |

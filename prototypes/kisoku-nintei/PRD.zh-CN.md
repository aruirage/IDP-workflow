# NeosAI 日本保险 IDP 工作流配置— 产品需求文档（PRD）

## 第 6 章｜功能详细说明

### 6.01 业务场景设置

业务场景设置包含三类场景：新建场景、编辑已发布场景、复制已有场景。三类场景共用三步配置和发布校验。

#### 功能点

1. 新建场景
  - 配置人员从场景列表创建空草稿。
  - 系统生成业务场景 ID，状态为草稿。
  - 配置人员必须完成账票集合、工作流画布、输出字段和工作流测试后才能发布。
2. 编辑已发布场景
  - 配置人员可直接打开已发布场景继续编辑。
  - 保存或修改工作流后，场景状态回到草稿，需重新测试并发布。
3. 复制已有场景
  - 系统复制基础信息、账票集合、主副账票关系、工作流定义和输出字段。
  - 复制后状态为待确认。
  - 配置人员必须复核被复制的账票、字段和规则引用；存在失效引用时阻断发布。
4. 账票关系配置
  - 配置业务场景名称、说明、账票集合、主账票、主字段、关联账票字段关系和匹配策略。
  - 账票类型候选来自账票类型设置。
  - 主账票必须且只能有一个；关联账票可有多个。
  - 每个关联账票至少配置一组字段关系。
5. 工作流入口
  - 配置人员从第 2 步进入工作流画布。
  - 工作流画布读取第 1 步账票集合和主副账票关系，限制节点可选账票、字段和规则范围。
  - 画布保存后同步可发布状态；若保存内容影响执行逻辑，则将工作流测试结论置为未测试。
6. 输出字段
  - 配置人员选择对外输出字段和顺序。
  - 字段来源包括 OCR 抽出字段、标准字段（第 2 步配置数据映射节点时可用）、工作流结果字段。
  - 输出字段被导出和对外 API 消费。
7. 发布
  - 发布前校验三步配置、工作流测试、依赖引用和输出字段。
  - 发布成功后场景状态变为已发布，新案件使用当前配置。

### 6.02 工作流画布与可编排节点

工作流画布包含三类场景：编辑画布、配置节点、执行测试。画布只编排案件集约完成后的处理段，不编排上传、案件集约和导出。

#### 功能点：

1. 画布编辑
  - 配置人员从业务场景设置进入画布。
  - 支持添加节点、插入节点、连接节点、删除节点、删除连线、自动整理。
  - 任何影响执行逻辑的修改都会把工作流测试结论置为未测试。
2. 节点选择器
  - 业务 Agent：AI 前处理、OCR 抽出、数据映射、AI 检证。
  - 控制节点：条件判断、人工确认、通知。
  - 扩展节点：自定义函数。
  - 端点：开始、结束。
  - 通过画布右下角工具栏「ノードを追加」或节点连接点唤起浮层选择器。
3. 节点配置
  - 选中节点后，在右侧节点编辑面板配置参数；节点编辑面板标题旁 ⓘ 悬停展示节点说明，画布节点标题下展示摘要。
  - 处理节点底部以折叠区展示「出力変数」（输出变量）；条件判断、通知等通过变量级联选择器引用上游输出变量。
  - 账票类别设定、数据映射、AI 检证节点不在节点编辑面板内编辑规则明细，仅提供跳转至对应设置页。
4. 上游变量引用
  - 条件判断、通知变量插入、自定义函数入参等，只可引用当前路径可达的所有上游输出变量。
5. 分支与出口
  - 普通节点默认一个完成出口。
  - 条件判断节点支持 IF、ELIF、ELSE。
  - 人工确认节点固定三种结果分支：完成、补件、案件终止。
6. 待办生成
  - AI 前处理节点可生成前处理确认待办。
  - OCR 抽出节点可生成 OCR 抽出确认待办。
  - AI 检证节点可生成 AI 检证确认待办。
  - 人工确认节点须接在前处理、OCR 或 AI 检证之后（如中间插入条件节点，自动穿透检测上游最近的业务agent节点）；上游不合法时测试标记失败。
7. 调用逻辑
  - 案件集约完成后，齐套新建案件时启动工作流实例；补件归入、补件直挂或后续文件归入时续跑已有实例；处理中止后重新执行时绑定当前生效工作流版本启动新实例。
  - 工作流按顺序执行每个节点，节点内按文件并行处理，处理完后进入下一个工作流节点。
  - AI 前处理和 OCR 抽出按文件执行，并向上汇聚节点进度。
  - 数据映射、AI 检证、条件判断、通知、自定义函数按案件执行。
  - 数据映射与 AI 检证为案件级节点，进入节点业务逻辑前须先通过批次文件到齐校验（见「案件级节点入口校验」）；未通过时节点保持等待，不执行映射或检证。
  - 人工确认节点生成待办后暂停所在分支，待办办结后按动作出口继续。
  - 到达结束节点时当前分支结束；所有活跃分支均结束且无未办结人工确认待办时，工作流完结并触发导出待办。
8. 工作流测试
  - 测试对象是当前工作流草稿与按业务场景保存的集约済み测试用例；不执行上传与案件集约，不生成正式案件、正式待办。
  - 系统内置一组标准已集约完成的测试用例；用户可上传 ZIP/JSON 覆盖并持久化到当前业务场景，下次打开自动沿用；也可恢复内置默认。
  - 点击「执行测试」后，系统先校验测试用例；通过后沿从开始节点可达的路径依次模拟各节点，并在右侧时间线逐步展示执行状态与结果。孤立节点、不可达节点不在时间线列出。
  - 画布结构、节点配置、变量引用等校验在模拟执行过程中按节点判定；可达路径上的失败节点在时间线标记为失败，标题下方展示执行摘要，再下一行展示可读失败原因；孤立/不可达等结构问题以画布高亮与发布门禁为主。
  - 模拟执行到达结束节点且无失败节点时为测试成功；否则为测试失败。结论回写业务场景设置中的最近测试结论与可发布状态。

#### 案件级节点入口校验

此校验由工作流引擎在运行时执行；配置端数据映射与 AI 检证节点编辑面板不展示校验项，仅在本节定义规则。

数据映射与 AI 检证按案件执行，依赖本案全部逻辑文件的上游处理结果。工作流引擎在节点被调度时、执行业务逻辑之前，先校验本实例当前文件范围内是否全部到齐；校验通过后才进入节点执行，未通过则节点保持等待，待文件状态变更后自动重试。

校验范围：本工作流实例当前待处理文件范围内的全部逻辑文件（含续跑、补件归入、后续文件归入后新增的文件）。

到齐判定以文件的两段状态为准：集约关联状态须为关联成功；工作流内处理状态须满足下表。仍在等待处理或处理中的文件视为未到齐。


| 校验项        | 数据映射入口 | AI 检证入口          | 判定说明                           |
| ---------- | ------ | ---------------- | ------------------------------ |
| 关联成功       | 是      | 是                | 文件已纳入本案，非等待关联或关联失败             |
| 前处理终态      | 是      | 是                | 每份文件的前处理已结束（成功、警告、失败或跳过均可）     |
| OCR 抽出终态   | 是      | 是                | 每份文件的 OCR 抽出已结束（成功、警告、失败或跳过均可） |
| 文件级确认待办已办结 | 是      | 是                | 不存在未办结的前处理确认或 OCR 抽出确认待办       |
| 数据映射已完成    | —      | 是（仅当路径可达数据映射节点时） | 上游数据映射节点已执行完成；无数据映射节点时跳过此项     |


未通过时的引擎行为：

1. 节点进度标记为等待，不写入映射或检证输出变量。
2. 已有文件继续按上游节点并行处理；新批文件归入后纳入同一校验范围。
3. 每次文件状态或待办状态变更后重新评估；全部到齐后自动进入节点业务逻辑。
4. 案件详情与节点进度界面展示等待原因（未到齐文件数或待办未办结），不生成映射/检证失败结论。

测试弹窗为同步模拟，默认假定测试用例内文件均已到齐，不模拟跨批次等待；正式运行由引擎执行上述校验。

#### 变量作用域与引用规则

工作流变量按作用域分为三类，与节点编辑面板输出变量折叠区、条件判断变量选择器、通知变量插入中的展示一致。


| 作用域   | 界面标注  | 含义                     | 典型引用路径                   |
| ----- | ----- | ---------------------- | ------------------------ |
| 案件级   | 案件    | 本案一次节点执行后的聚合结果，不区分单文件  | `{节点变量名.case.项目键}`       |
| 文件级   | ファイル  | 本案每个逻辑文件各一份，按文件 ID 索引  | `{节点变量名.files[].项目键}`    |
| 账票类型级 | 帳票タイプ | 与 Step1 关联账票类型绑定的清单或枚举 | `{节点变量名.docTypes[].项目键}` |


引用规则：

1. 节点变量名由引擎按节点在画布上的唯一标识生成；配置端只读展示，不可改。
2. 条件判断、通知变量插入、自定义函数入参仅可选择当前路径可达的上游节点输出变量。
3. 对象、数组类型变量在条件表达式中须选择子字段、件数或是否为空等派生项后再比较。
4. 上游节点删除、账票类型移除、映射规则或检证规则失效时，下游引用标记为错误并阻断发布。
5. 文件级变量在导出配置中须明确展开方式（按文件、按账票类型聚合等）。

界面文案：出力変数；引用格式说明——後続ノード・IF/ELSE 条件で `{ノード変数名.項目}` 形式で指定します。

#### 各节点输出变量

配置端在节点编辑面板底部「出力変数」折叠区只读展示下列变量。展示格式：第一行变量键 + 类型标签；第二行界面文案（日语）。不展示取值说明。

精简原则：只暴露下游条件判断、通知、导出、人工确认路由会引用的变量。与状态枚举重复的计数字段、引擎内部字段（任务 ID、缓冲计时、文件 ID 列表等）不列入配置端目录。各业务 Agent 节点统一保留「节点状态」枚举 +「人工确认必要」布尔，供后续分支配线。

带「配置影响」的项表示：开关关闭、对象账票未选中或规则未命中时，该项可能为空、为 skipped，或不写入。

#### 工作流测试流程

测试在测试弹窗内完成。案件集约不在测试范围内；测试输入为内置标准已集约完成的用例，或用户上传并已保存的 ZIP/JSON 快照。配置阶段无法从案件一览选取真实案件，因案件仅在业务场景发布并受理后才会产生。

弹窗布局：

- 左侧：测试用例与关联文件清单（可折叠）、ZIP/JSON 上传；未选中节点时提示点击右侧节点查看详情；选中后在左侧下方展示该节点的模拟输入、输出与配置问题列表。
- 右侧：Workflow 実行时间线，仅展示从开始节点沿有向边可达路径上的节点，按顺序展示执行状态、执行摘要与失败原因。不在可达路径上的孤立节点、不可达节点不在时间线列出，改由画布编辑软提示与发布门禁处理。

执行顺序（用户只点击一次「执行测试」，系统内部按序执行）：

1. 测试用例校验：检查案件编号、案件名、文件清单；无效则提示并中止，不启动时间线。
2. 节点模拟执行：沿从开始节点可达的路径依次模拟；时间线逐步高亮当前节点；人工确认节点标记为跳过；环路径节点在摘要中标注；各层校验项在对应节点模拟时判定，失败则该节点状态为失败并写入错误原因。
3. 结论汇总：扫描全画布结构（含孤立/不可达节点）；存在任一阻断项、模拟失败节点或未到达结束节点时为测试失败；仅无失败节点且到达结束节点时为测试成功。结论回写业务场景设置中的最近测试结论与可发布状态。

测试失败展示：失败节点与失败原因以右侧时间线为主。每个节点块含节点名、状态、执行摘要；失败时在摘要下方展示错误原因（errorReason）。用户点击该节点后，左侧详情区补充节点输入、输出与问题说明。弹窗顶部展示总体结论（成功/失敗/終了未到達）、处理时间与通过统计；时间线或顶部可提供「エラー箇所へ移動」跳转画布对应节点（高亮后自动消退）。

界面文案：测试弹窗标题——Workflow テスト；时间线标题——Workflow 実行；执行中按钮文案——実行中…；成功提示——Workflow テストを実行しました；上传成功——テストデータを更新しました：{文件名}；恢复默认——初期値に戻す。

- ZIP 须能解析为集约済み元数据结构；无法解析时提示格式错误并拒绝覆盖用例。

#### 测试校验

校验按时机分为四类：画布编辑时的连线/保存软提示；模拟执行中按节点逐项校验并在右侧时间线展示结果；发布前的全量门禁；测试用例层在点击执行前校验。测试弹窗不设独立的错误清单侧栏；阻断项与警告项挂载到时间线可达节点、弹窗顶部结论，或对孤立/不可达节点做画布高亮。

界面文案：テスト用データ；Workflow 実行；エラー箇所へ移動

| 校验项   | 阻断  | 判定条件                                    | 用户提示 / 处理                                  |
| ----- | --- | --------------------------------------- | ------------------------------------------ |
| 开始节点  | 是   | 开始节点多于 1 个（系统保证至少存在 1 个开始节点）            | 时间线开始节点失败：開始ノードが複数あります                     |
| 结束节点  | 是   | 无结束节点                                   | 模拟执行后顶部「終了未到達」；时间线路径末端无结束节点                |

| 结束不可达 | 是   | 从开始节点无法到达任一结束节点                         | 顶部「終了未到達」；相关断点节点时间线展示原因                    |

| 并行死胡同 | 是   | 并行分支末端无结束节点且无法汇入主路径                     | 时间线分支末端节点失败：並列分岐が終了に到達しません                 |

错误展示约定：

1. 测试用例无效：仅提示校验文案，不启动时间线。
2. 模拟执行失败：以右侧时间线为主。节点块顺序为：节点名与状态 → 执行摘要（一行）→ 失败原因（仅失败时，红色文案）。用户点击节点后，左侧补充输入输出详情；不在左侧重复堆叠整条节点列表。
3. 发布阻断：可汇总最近测试失败节点与发布门禁项；画布跳转沿用「エラー箇所へ移動」。

| 孤立节点     | 节点无任何入边与出边                  | 测试失败；画布高亮；不在测试时间线展示           |
| 不可达节点    | 从开始节点无法到达                   | 测试失败；画布高亮；不在测试时间线展示           |
| 分支无落点    | 条件判断出口未连接；或人工确认三类出口均未连接 | 时间线该节点失败；画布连线时软提示             |
| 回流不闭环    | 回流环无法到达结束节点                 | 时间线环上节点失败                     |
| 变量引用失效   | 上游节点删除或变量不可达                | 时间线引用节点失败；字段标记失效              |
| 人工确认上游错误 | 人工确认未接在可产生待办的节点之后           | 时间线该节点失败，摘要下展示说明              |
| 通知接收人为空  | 通知节点未配置收件人                  | 时间线通知节点失败                     |
| 并行分支未完结  | 仍有活跃分支未到达结束节点               | 工作流不完结，不生成导出待办                |
| 案件级节点等待  | 数据映射或 AI 检证入口文件未到齐          | 节点进度为等待；文件到齐后自动进入；案件详情展示等待原因  |
| 测试数据污染   | 测试生成正式案件、待办或站内信             | 禁止写入正式列表，仅保留测试记录              |
| 删除开始节点   | 用户试图删除开始节点                  | 「開始ノードは削除できません。」              |
| 节点删除成功   | 删除非开始节点成功                   | 「ノードを削除しました」                  |
| 连接成功     | 节点连线成功                      | 「接続しました」或「回流接続しました」（回流连线时）    |
| 连接提示     | 拖拽连线时                       | 「接続先ノードの入力ポートへドラッグしてください」     |
| 节点交换限制   | 非主流程相邻节点交换                  | 「メインフロー上の隣接ノードのみ入れ替えできます」     |
| 重置确认     | 用户重置当前步骤配置                  | 确认弹窗「このステップの変更を破棄してリセットしますか？」 |

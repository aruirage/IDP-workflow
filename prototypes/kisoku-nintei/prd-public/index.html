6.01 业务场景设置

三步向导：Step1 案件集约规则，Step2 跳转工作流画布，Step3 输出字段与发布。新建、编辑已发布、复制三类场景共用流程；影响执行逻辑的变更将工作流测试结论置为未测试。

功能点

  1. 场景列表：新建、复制、重命名、删除、关键词检索；复制后状态为待确认，须复核失效引用。
  2. Step1：配置名称、说明、账票集合（1～20）、主账票与主字段、关联账票字段关系、匹配策略；支持关联检查与 AI 自动关联。
  3. Step2：进入工作流画布；画布读取 Step1 账票与关系，限制节点可选范围。
  4. Step3：勾选并排序输出字段（OCR 字段、标准字段、工作流结果字段等）。
  5. 发布：三步校验通过且工作流测试成功后方可发布；新案件使用发布后配置。

6.02 工作流画布与可编排节点

画布只编排案件集约完成后的处理段；不编排上传、案件集约、导出。节点规则明细在账票类型、数据映射、AI 检证等设置页维护，画布仅配置外壳、开关、引用与分支。

功能点

  1. 画布编辑：添加/插入/连接/删除节点与连线、自动整列；逻辑变更置测试为未测试。节点标题固定为类型名（如「条件判断」），不拼接上下文名。
  2. 节点选择器（浮层）：业务 Agent（AI 前处理、OCR 抽出、数据映射、AI 检证）、控制（条件判断、人工确认、通知）、扩展（自定义函数）、端点（开始、结束）。入口：右下角「ノードを追加」或连接点。
  3. 节点编辑面板：按类型展示参数；底部折叠「出力変数」只读展示（变量键 + 类型 + 界面文案，不展示取值说明）。数据映射、AI 检证仅提供跳转设置页，不在面板内编规则。
  4. 变量引用：条件、通知、自定义函数入参仅可选当前路径可达上游输出变量；失效阻断发布。
  5. 分支：条件判断 IF/ELIF/ELSE（机器规则分支）；人工确认固定完成、补件、案件终止三出口（将人工裁量揉进节点内，审查者选出口即路由，无需再写条件表达式）。
  6. 人工确认：确认对象由上游最近业务 Agent 自动判定；须接在前处理/OCR/AI 检证之后（穿透条件节点检测），否则测试失败。三分支直连下游，替代「人工确认后再接条件判断」的写法。只需连接实际用到的出口（可只接一条）；未接下游的出口自动标为「不要」，校验不要求三分支全部接线，但至少需接一条。
  7. 工作流测试：对象=当前草稿 + 内置标准集约済み用例；不跑上传/集约，不写正式案件/待办。弹窗左侧只读用例与节点详情，右侧「Workflow 実行」时间线展示可达路径执行结果。

案件级节点入口校验

数据映射与 AI 检证为案件级节点；引擎在调度时、执行业务逻辑前校验本案全部逻辑文件到齐。配置端不在节点面板展示校验项。

校验项数据映射AI 检证说明
集约关联成功是是见总 PRD 文件状态
文件级确认待办已办结是是前处理/OCR 确认待办
数据映射已完成—路径可达时无映射节点则跳过

未通过：节点进度为等待，文件状态变更后自动重试。测试弹窗默认假定用例内文件已到齐。

工作流变量（总览）

作用域仅分三类：案件、文件、账票类型级。各节点输出变量、条件选择器、出力変数面板均按此标注。

作用域界面标注路径格式出力変数面板条件选择器
案件案件{节点}.case.{字段}列出各 case 字段可选
文件ファイル{节点}.files[]仅列 files[] 一项不可选
账票类型级帳票タイプ{start}.docTypes[]仅开始节点列出可选

补充说明：

  • 控制节点（条件判断 / 通知 / 结束）仅输出案件级，无 files[]。
  • files[] 元素结构（id、name、url 等）见下文专表；为运行时约定，非出力変数面板逐条清单。分类结果在 classificationResult，OCR 值在 ocrFields。
  • 标准字段（case.standardFields）归属案件，仅数据映射节点产出；条件节点三级选择「映射节点 → 標準フィールド → 字段名」。
  • 账票字段（docTypes.{类型}.{字段}）归属账票类型级下的 Step1 字段引用，不是节点输出；条件选择器按 Step1 账票类别模板加载，运行时从模板取项目。

通用规则：仅选可达上游变量；上游删除或规则失效标记错误并阻断发布。文件级（数组）不可直接做条件比较。

人工分岐：是否进入人工确认，由画布 条件节点 → 人工确认节点 决定；条件读取各 Agent 的事实变量（见下表）。

节点条件分岔常用变量
前处理preprocessStatus
OCRlowConfidenceFieldCount、ocrStatus
AI 检证verifyStatus、各模块 *Status、missingDocuments / missingFields、isException

典型接法：OCR lowConfidenceFieldCount > 0 → 人工确认；AI 检证 requiredDocumentStatus = missing → 人工确认（补件）→ 补件通知。

人工确认节点

固定三出口（完成 / 補件 / 案件終止），把条件判断揉进人工确认节点：审查者在待办里选一个 action，引擎沿对应连线直接往下走，不必再写 IF/ELSE 表达式。confirmAction 仅作记录与多路径汇合后的引用。

出口接线规则：配置时只连接业务需要的出口（常见如仅接「完成」一条）；未接下游的出口自动视为「不要」，不要求三分支全部接线，但至少需接一条。运行时待办仅展示已接线的 action 供审查者选择。

工作流测试

输入：仅内置标准集约済み用例，打开弹窗即加载，只读；不支持上传、选案或弹窗内编辑。

执行：用户点一次「テスト実行」→ 用例校验 → 沿可达路径模拟（人工确认标记跳过）→ 全画布扫描结论。成功=无失败节点且到达结束；否则失败。结论回写业务场景测试状态。

校验与展示（按层归类；失败以右侧时间线为主，孤立/不可达画布高亮）：

层级代表阻断项用户提示（日语，节选）
用例案件名空、文件清单空案件名を入力してください 等
结构无结束、孤立、不可达、分支未接、回流不闭环終了未到達、接続のないノードがあります 等
节点配置前处理/OCR/检证全关、条件空、人工确认上游非法、通知无收件人、函数不完整前処理オプションを1件以上… 等
变量引用不可达、不存在、类型不匹配変数が見つかりません 等
依赖账票/模板/映射/检证/导出字段失效发布阻断或时间线节点失败
发布门禁与上同规则集 + 测试结论非成功ワークフローテストを完了してください

测试结论映射：任一阻断或模拟失败→测试失败→不可发布；仅警告且到达结束→可发布。

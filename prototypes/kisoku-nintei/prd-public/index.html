6.01 业务场景设置

6.01.1 概述

四步向导：Step1 案件集约规则，Step2 工作流画布，Step3 通知设置，Step4 输出字段与发布。新建、编辑已发布、复制场景共用流程。案件场景只展示一个主状态，不展示分步状态。各步骤保存只写草稿配置；只有发布生成正式版本号。

6.01.2 功能点

  1. 场景列表：新建、复制、重命名、删除、关键词检索；复制后进入 Step1 继续编辑。
  2. Step1：配置名称、说明、账票集合（1～20）、主账票与主字段、关联账票字段关系、匹配策略；支持关联检查与 AI 自动关联。
  3. Step2：进入工作流画布；画布读取 Step1 账票与关系，限制节点可选范围。工具栏提供工作流测试；测试通过才允许进入 Step3。
  4. Step3：按工作流节点配置内置通知。配置对象、主题、内容和可插入变量；页面不展示触发条件。处理节点按节点专属 Status / Result 自动判断；人工确认按完成、补件、案件终止三出口事件触发。
  5. Step4：从 OCR 抽出账票类型中勾选要导出的字段（允许 0）。点保存固化勾选；0 字段也可以保存，表示用户确认本场景不导出 OCR 字段。
  6. 发布：场景须为可发布，且 Step3、Step4 的草稿配置均已保存有效。发布时生成新版本号；新案件只读取已发布版本。

保存与版本号：每一步保存只写草稿配置，不生成版本号，也不覆盖其他步骤已保存内容。Step1 保存只写场景基础与集约规则；Step2 保存只写画布、节点、连线和变量引用；Step3 保存只写通知配置；Step4 保存只写输出配置。发布读取 Step1～Step4 当前已保存草稿，生成一个新的正式版本号。

Step3：配置节点内置通知，不设启用/停用开关；配置存在即启用，删除配置即停用。点击「保存」或「次へ」都会保存当前通知配置；其中「次へ」在保存后直接进入 Step4。通知配置不重验工作流。

Step4：勾选要导出的 OCR 字段后点「保存」即可；允许 0 字段。有候选时可以不勾选任何字段，但必须保存一次，表示确认空输出。保存不改写 Step1～Step3，也不重验工作流。

已发布后再编辑：打开已发布场景并修改任一步时，系统从已发布版本派生草稿编辑态。保存后重新打开，继续看到已保存的草稿改动，不需要重新改；正式案件仍使用上一个已发布版本。只有再次通过 Step2 测试、保存 Step3/Step4，并点击发布后，才生成新版本并替换后续新案件使用的版本。

6.01.6 Step1～Step4 校验说明

Step1 提供关联检查；Step2 提供工作流测试，并作为进入 Step3 的门禁。Step3 没有测试，只保存通知配置；Step4 没有测试，只保存输出勾选。终了结构与到达在 Step2 测试内检查。公开时：须处于可发布主状态。

6.02 工作流画布与可编排节点

6.02.1 概述

画布只编排案件集约完成之后的处理段；不编排上传、案件集约、导出。节点规则在账票类型、数据映射、AI 检证等设置页维护，画布配置外壳、开关、引用与分支。

  • 各节点变量统一说明出力、消费路径、允许取值和可用运算符。
  • 配置端「出力変数」不提供折叠按钮；列表行只显示裸变量键 + 类型；点圆形 i 查看中文变量名和取值范围。i 气泡标签用日文（変数名 / 取値範囲 / 例），不展示长说明；files[] 只附 1 条短 JSON 示例。所有节点都直接平铺展示，不再显示案件/文件/账票类型分组标题。
  • 画布节点不再展示配置摘要行；节点卡片只显示类型名（条件/人工确认额外显示分支结构）。

6.02.2 功能点

  1. 画布编辑：添加/插入/连接/删除节点与连线、自动整列；逻辑变更置测试为未测试。节点标题固定为类型名。
  2. 节点选择器：业务 Agent（前处理、OCR、映射、检证）、控制（条件、人工确认）、扩展（自定义函数）、端点（开始、结束）。
  3. 节点编辑面板：按类型展示参数；底部「出力変数」不折叠，只读展示变量键 + 类型；中文变量名、取值范围、files[] 短示例在 i 中。映射、检证仅跳转设置页。
  4. 变量引用：条件仅选可达上游变量；自定义函数固定读取开始节点 files[]。会破坏既有引用的保存阻断，发布前仍发现不可达引用时发布阻断。
  5. 分支：条件 IF/ELIF/ELSE；人工确认三出口（完成/补件/案件终止），审查者选出口即路由。
  6. 人工确认：须接在前处理/OCR/检证之后；只接需要的出口，至少一条。
  7. 工作流测试：内置集约完成用例；不跑上传/集约，不写正式案件/待办。测试成功后允许进入 Step3；测试失败阻断进入 Step3。
  8. Step1 派生 schema：Step2 的账票类别和 OCR 字段只来自 Step1 关联账票集合；Step2 不能新增、删除或重命名账票类别和 OCR 字段。
  9. 说明图标统一为圆形 i；画布不再提供「全部折叠节点」入口。复制场景后不展示 Step1「要確認」引用检查横幅。

6.02.8 工作流测试

Step2 画布用工作流测试证明「当前画布配置可跑通」。测试通过是进入 Step3 的门禁；它不是正式跑件：不创建正式案件、不写真实待办。内置样例是测试输入前提，不作为用户配置测试项；本节只补「怎么操作、成功/失败写什么、弹什么提示」。

功能点：

  1. 配置人员在 Step2 打开测试弹窗，确认内置样例后执行测试。
  2. 系统按「用例 → 结构 → 连线 → 节点配置 → 变量引用 → 字段选择 → 依赖 → 模拟执行 → 发布门禁」顺序校验；前一层阻断则不进入后一层。结束节点有无、无出边、可达结束等归结构/模拟维度，不在测试外再挂一层。
  3. 模拟执行按样例字段求值后的命中路径生成时间线（条件走真分支，人工确认跳过真人动作后按已连接出口继续）；任一节点失败立即停止，后续节点保持未执行。
  4. 全部规则通过：写测试结论=成功，场景状态置为可发布。
  5. 任一失败（含未配置结束、不可达结束）：写测试结论=失败，场景保持草稿（非已发布时），阻断进入 Step3。
  6. 人工确认节点在测试中跳过真实确认动作（不等人、不建真实待办），校验上下文与出口后继续下游，以便测通后续节点。正式运行仍须真人确认。
